## Öğrenim Hedefleri

Mekansal verinin en yaygın kaynaklarından biri CSV dosyalarıdır; konum bilgisi genellikle enlem/boylam sütunları biçiminde saklanır. Bu ders, metin tabanlı koordinat değerlerini PostGIS geometry sütununa dönüştürmenin adımlarını ve yaygın hata senaryolarını ele almaktadır.

- CSV verisini PGlite'ta bir tabloya aktarmak
- Enlem/boylam sütunlarından geometry sütunu oluşturmak
- Temiz veri yükleme adımlarını açıklamak

---

## Tarayıcıda Veri Yükleme

Sunucu tarafı araçları (`COPY`, `psql \copy`, `shp2pgsql`) bu sandbox ortamında doğrudan kullanılamaz. Kullanılabilecek alternatifler:

1. **INSERT VALUES** — küçük test verisi için
2. **Programatik INSERT** — JavaScript'ten PGlite API ile
3. **Staging tablosu deseni** — veriyi metin olarak yükle, ardından geometry'ye dönüştür

Bu ders, SQL `INSERT` ifadeleriyle bir CSV dosyasını simüle ederek nokta tablosu oluşturma akışını göstermektedir.

### Staging'den geometry tablosuna aktarım

<RunnableBlock
  sql={`-- Hedef tablo: geometry sütunu ile
CREATE TABLE IF NOT EXISTS noktalar (
  id   SERIAL PRIMARY KEY,
  ad   TEXT NOT NULL,
  geom GEOMETRY(POINT, 4326)
);

-- Staging'den dönüştürerek aktar
INSERT INTO noktalar (ad, geom)
SELECT
  ad,
  ST_SetSRID(
    ST_MakePoint(
      lon_text::FLOAT,
      lat_text::FLOAT
    ),
    4326
  )
FROM csv_staging
ON CONFLICT DO NOTHING;

SELECT id, ad, ST_AsText(geom) AS wkt FROM noktalar;`}
  label="Staging → geometry tablosu"
  description="Staging tablosundaki metin koordinatları ::FLOAT cast ile sayıya çevrilir; ST_MakePoint(lon, lat) ile POINT geometrisi oluşturulur; ST_SetSRID SRID=4326 atar. ON CONFLICT DO NOTHING çift yükleme girişimlerini sessizce atlar. Sonuç olarak hedef tablonun WKT gösterimi doğrulanır."
/>

### Nokta tablosunu haritada görüntüle

## Yaygın Hatalar

1. **Virgüllü ondalık sayı.** Bazı CSV dosyalarında `28,979` (virgül) biçiminde koordinatlar bulunur. PostgreSQL nokta (`.`) bekler. `REPLACE(lon_text, ',', '.')::FLOAT` ifadesiyle düzeltilmelidir.
2. **Enlem/boylam sırası.** CSV'de `lat, lon` sırası kullanılabilir; ancak `ST_MakePoint(lon, lat)` sırası bekler. Sütun adları dikkatle kontrol edilmelidir.
3. **NULL koordinatlar.** `lon_text` veya `lat_text` NULL olduğunda `ST_MakePoint` NULL döndürür. `WHERE lon_text IS NOT NULL AND lat_text IS NOT NULL` koşuluyla bu satırlar filtrelenmelidir.
