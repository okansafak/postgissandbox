## Öğrenim Hedefleri

GeoJSON, web tabanlı harita uygulamalarında yaygın olarak kullanılan standart bir coğrafi veri formatıdır. PostGIS, GeoJSON metnini doğrudan geometry'ye dönüştürebilir; bu özellik, harici veri kaynaklarından veri yüklemede pratik bir alternatif sunar.

- `ST_GeomFromGeoJSON` ile GeoJSON metnini geometry'ye dönüştürmek
- GeoJSON özelliklerini tablo sütunlarına eşlemek
- `json_array_elements` ile GeoJSON FeatureCollection'ı satırlara açmak

---

## GeoJSON Yükleme

GeoJSON, web harita ekosisteminin standart formatıdır. PostGIS `ST_GeomFromGeoJSON` fonksiyonu bu formatı doğrudan ayrıştırır:

```sql
ST_GeomFromGeoJSON('{"type":"Point","coordinates":[28.979,41.015]}')
```

Gerçek senaryolarda GeoJSON `properties` nesnesinden attribute sütunları çıkarılır ve geometry ile birlikte tabloya yazılır.

---

## Uygulama Örneği

### Tek Feature dönüşümü

<RunnableBlock
  sql={`SELECT
  ST_AsText(
    ST_SetSRID(
      ST_GeomFromGeoJSON('{"type":"Point","coordinates":[28.979,41.015]}'),

  ) AS wkt,
  ST_SRID(
    ST_GeomFromGeoJSON('{"type":"Point","coordinates":[28.979,41.015]}')
  ) AS srid_otomatik;`}

ST_AsText sonucu insan tarafından okunabilir WKT formatında gösterir."

### FeatureCollection yükleme

<RunnableBlock
  sql={`CREATE TABLE IF NOT EXISTS geojson_noktalar (
  id   SERIAL PRIMARY KEY,
  ad   TEXT,
  tur  TEXT,
  geom GEOMETRY(POINT, 4326)
);

-- JSON içinden feature'ları çıkar
WITH fc AS (
  SELECT '{"type":"FeatureCollection","features":[
    {"type":"Feature","properties":{"ad":"Galata Kulesi","tur":"tarihi"},"geometry":{"type":"Point","coordinates":[28.974,41.026]}},
    {"type":"Feature","properties":{"ad":"Topkapı Sarayı","tur":"müze"},"geometry":{"type":"Point","coordinates":[28.983,41.011]}},
    {"type":"Feature","properties":{"ad":"Kız Kulesi","tur":"tarihi"},"geometry":{"type":"Point","coordinates":[29.004,41.021]}}
  ]}'::json AS fc
)
INSERT INTO geojson_noktalar (ad, tur, geom)
SELECT
  feat->'properties'->>'ad'                                         AS ad,
  feat->'properties'->>'tur'                                        AS tur,
  ST_SetSRID(ST_GeomFromGeoJSON(feat->>'geometry'), 4326)           AS geom
FROM fc,
     json_array_elements(fc->'features') AS feat(feat)
ON CONFLICT DO NOTHING;

SELECT id, ad, tur, ST_AsText(geom) AS wkt FROM geojson_noktalar;`}
  label="FeatureCollection → tablo"
  description="json_array_elements FeatureCollection içindeki features dizisini tek tek satırlara açar. Her satır için ->'properties' JSON nesnesi, ->>'properties'->>'ad' ise metin değer döndürür. feat->>'geometry' geometri bloğunu metin olarak alır ve ST_GeomFromGeoJSON dönüştürür. ON CONFLICT DO NOTHING aynı sorgunun tekrar çalıştırılmasında çakışmaları önler."
/>

### GeoJSON noktalarını haritada görüntüle

## Pratik Uygulama

<RunnableBlock
  sql={`SELECT
  ST_AsText(ST_GeomFromGeoJSON(
    '{"type":"Polygon","coordinates":[[[28.9,41.0],[29.1,41.0],[29.1,41.1],[28.9,41.1],[28.9,41.0]]]}'
  )) AS polygon_wkt,
  ROUND(ST_Area(ST_GeomFromGeoJSON(
    '{"type":"Polygon","coordinates":[[[28.9,41.0],[29.1,41.0],[29.1,41.1],[28.9,41.1],[28.9,41.0]]]}')
  ::geography)::numeric / 1000000, 1) AS alan_km2;`}
  label="GeoJSON Polygon yükle"
  description="GeoJSON formatındaki bir Polygon geometrisi ST_GeomFromGeoJSON ile dönüştürülür ve WKT biçiminde gösterilir. ST_Area ::geography cast ile elipsoid üzerinde m² cinsinden alan hesaplar; 1.000.000'a bölüm km²'ye çevirir. Bu örnek ST_GeomFromGeoJSON'un nokta dışında Polygon gibi karmaşık geometrileri de desteklediğini gösterir."
/>

## Yaygın Hatalar

1. **`->`  ile `->>`  farkı.** `feat->'geometry'` JSON nesnesi döndürür; `feat->>'geometry'` metin döndürür. `ST_GeomFromGeoJSON` metin beklediğinden `->>` operatörü kullanılmalıdır.
2. **İç içe `properties`.** Özellik değerlerine erişmek için `feat->'properties'->>'ad'` ifadesi gereklidir; `feat->>'ad'` feature kök düzeyinde arar ve NULL döndürür.
3. **Karışık geometry tipleri.** FeatureCollection'da Point, LineString ve Polygon bir arada bulunabilir. Bu durumda `GEOMETRY(POINT, 4326)` yerine kısıtsız `GEOMETRY` kolonu kullanılmalı ya da geometri tipi önceden filtrelenmelidir.
