## ogr2ogr / shp2pgsql — Kavramsal Genel Bakış

Komut satırı CBS araçları, büyük ölçekli veri yüklemede ve format dönüşümünde endüstri standardıdır. Bu ders, bu araçların hangi sorunları çözdüğünü, nasıl kullanıldığını ve PGlite sandbox ortamında hangi alternatiflerin geçerli olduğunu ele almaktadır.

---

## Öğrenim Hedefleri

- `ogr2ogr`'ın desteklediği formatları tanımak
- `shp2pgsql` ile shapefile yükleme pipeline'ını açıklamak
- PGlite'ta bu araçların neden çalışmadığını ve SQL alternatiflerini bilmek

---

## ogr2ogr Nedir?

`ogr2ogr`, GDAL kütüphanesinin bir parçasıdır ve **150'den fazla coğrafi veri formatı** arasında dönüşüm gerçekleştirir.

```bash
# Shapefile → PostgreSQL
ogr2ogr -f "PostgreSQL" \
  PG:"host=localhost dbname=gisdb user=postgres" \
  turkiye_iller.shp \
  -nln public.iller \
  -t_srs EPSG:4326

# GeoJSON → PostgreSQL
ogr2ogr -f "PostgreSQL" \
  PG:"host=localhost dbname=gisdb" \
  Istanbul_mahalleler.geojson \
  -nln mahalleler

# PostGIS → GeoPackage (dışa aktarım)
ogr2ogr -f "GPKG" cikti.gpkg \
  PG:"host=localhost dbname=gisdb" \
  -sql "SELECT * FROM iller WHERE nufus > 1000000"
```

**Desteklenen formatlar (seçme):** Shapefile, GeoJSON, GeoPackage, KML, GML, CSV+WKT, DXF, FileGDB, OSM, WFS

---

## shp2pgsql Nedir?

`shp2pgsql`, PostGIS ile birlikte gelen bir araçtır. Shapefile'ı SQL INSERT ifadelerine dönüştürür ve doğrudan bir PostgreSQL veritabanına yükleyebilir.

```bash
# SQL dosyası oluştur
shp2pgsql -s 4326 -W UTF-8 il_sinirlari.shp public.iller > iller.sql

# Doğrudan yükle
shp2pgsql -s 4326 il_sinirlari.shp public.iller | psql -d gisdb

# Seçenekler
# -s: SRID
# -W: encoding (UTF-8, LATIN1…)
# -a: append (tabloya ekle, yeni oluşturma)
# -d: drop & recreate
# -p: yalnızca CREATE TABLE (veri yok)
```

---

## raster2pgsql

Raster veriler için `shp2pgsql`'in kardeşidir:

```bash
raster2pgsql -s 4326 -I -C dem.tif public.dem | psql -d gisdb
```

---

## PGlite'ta Neden Çalışmaz?

Bu araçlar işletim sistemi süreçleridir ve tarayıcı sandbox'ı dışında çalışır. PGlite bir WASM PostgreSQL olduğundan:

1. Dış süreç çağrısı yapılamaz
2. Dosya sistemi erişimi yoktur
3. `psql` bağlantısı kurulamaz

CREATE TABLE IF NOT EXISTS ornek_noktalar (
  id   SERIAL PRIMARY KEY,
  ad   TEXT,
  geom GEOMETRY(POINT, 4326)
);

INSERT INTO ornek_noktalar (ad, geom)
SELECT
  'Nokta ' || i,
  ST_SetSRID(ST_MakePoint(
    28.5 + random() * 1.0,
    40.8 + random() * 0.5
  ), 4326)
FROM generate_series(1, 5) i;

SELECT id, ad, ST_AsText(geom) AS konum
FROM ornek_noktalar;`}

-- Koordinat sütunlarından geometri üret

## Araç Karşılaştırması

| Araç | Platform | Kaynak format | Hedef |
|---|---|---|---|
| `ogr2ogr` | Komut satırı | 150+ format | 150+ format |
| `shp2pgsql` | Komut satırı | Shapefile | PostgreSQL |
| `raster2pgsql` | Komut satırı | GeoTIFF vb. | PostgreSQL |
| SQL + ST_* | Her ortam | CSV, JSON, WKT | PostgreSQL |

Gerçek projelerde `ogr2ogr` ile yükleme yapılır; bu sandbox'ta SQL alternatifleri aynı mantığı göstermektedir.
