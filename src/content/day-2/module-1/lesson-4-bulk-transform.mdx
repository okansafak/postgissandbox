## Öğrenim Hedefleri

Gerçek dünya veri setlerinde koordinat sistemleri çoğunlukla tutarsızdır: bazı satırlar SRID etiketsiz gelir, bazıları yanlış SRID ile etiketlenmiştir, bazıları farklı projeksiyon sistemlerinde yer alır. Bu ders, bir tablodaki tüm geometrileri SQL `UPDATE` ile toplu biçimde standart bir koordinat sistemine taşıma yöntemini ele almaktadır.

- Tüm bir tablodaki geometrileri toplu `UPDATE` ile dönüştürmek
- `ST_SetSRID` ve `ST_Transform` zincirini doğru sırada uygulamak
- `ANALYZE` komutunun sorgu planlayıcı için önemi

---

## Toplu Dönüşüm

Gerçek veri genellikle karışık gelir:
- Bazı satırlarda `SRID = -1` (etiketsiz)
- Bazılarında yanlış SRID etiketi
- Bazıları UTM'de, bazıları EPSG:4326'da

Çözüm: `UPDATE` + `ST_Transform(ST_SetSRID(geom, kaynak_srid), hedef_srid)` zinciri.

## Uygulama Örneği

### Karışık SRID'li tablo

<RunnableBlock
  sql={`CREATE TABLE IF NOT EXISTS karisik_noktalar (
  id   SERIAL PRIMARY KEY,
  ad   TEXT,
  geom GEOMETRY,
  kaynak_srid INT
);

INSERT INTO karisik_noktalar (ad, geom, kaynak_srid) VALUES
  -- Doğru 4326
  ('İstanbul', ST_SetSRID(ST_MakePoint(28.979,41.015),4326), 4326),
  -- SRID eksik
  ('Ankara',   ST_MakePoint(32.854,39.930),                  -1),
  -- UTM 36N koordinatları
  ('Koordinat A', ST_SetSRID(ST_MakePoint(500000,4400000),32636), 32636)
ON CONFLICT DO NOTHING;

SELECT id, ad, ST_SRID(geom) AS mevcut_srid, ST_AsText(geom) AS geom_text
FROM karisik_noktalar;`}
  label="Karışık SRID tablosu"
  description="Üç farklı SRID senaryosunu temsil eden bir tablo oluşturulur: EPSG:4326 ile doğru etiketlenmiş nokta, SRID eksik nokta (0 döner) ve UTM 36N koordinatları. ST_SRID mevcut SRID değerini döndürür; geom_text ham koordinat değerlerini gösterir. UTM koordinatları WGS84'te anlamsız büyük sayılar olarak görünür."
/>

### Toplu SRID düzeltmesi

<RunnableBlock
  sql={`-- Adım 1: SRID=0 olanları 4326 olarak etiketle
UPDATE karisik_noktalar
SET geom = ST_SetSRID(geom, 4326)
WHERE ST_SRID(geom) = 0;

-- Adım 2: UTM 36N olanları 4326'ya dönüştür
UPDATE karisik_noktalar
SET geom = ST_Transform(geom, 4326)
WHERE ST_SRID(geom) = 32636;

-- Sonucu kontrol et
SELECT id, ad, ST_SRID(geom) AS yeni_srid,
  ROUND(ST_X(geom)::numeric, 4) AS lon,
  ROUND(ST_Y(geom)::numeric, 4) AS lat
FROM karisik_noktalar;`}
  label="Toplu SRID düzeltmesi"
  description="İki aşamalı UPDATE: ilk UPDATE ST_SetSRID ile yalnızca etiket ekler — koordinat değerlerini değiştirmez. İkinci UPDATE ST_Transform ile UTM koordinatlarını WGS84'e matematiksel olarak dönüştürür. Sıralama önemlidir: ST_Transform kaynak SRID'i bilmek zorundadır, bu nedenle etiketleme önce yapılır. Sonuç olarak tüm satırlar SRID=4326 ile anlamlı lon/lat değerleri taşır."
/>

### Typed tabloya aktarım ve ANALYZE

<RunnableBlock
  sql={`-- Tüm geometriler 4326'ya döndükten sonra kısıtlama ekle
CREATE TABLE IF NOT EXISTS noktalar_typed (
  id   SERIAL PRIMARY KEY,
  ad   TEXT,
  geom GEOMETRY(POINT, 4326)
);

INSERT INTO noktalar_typed (ad, geom)
SELECT ad, geom::geometry(POINT, 4326)
FROM karisik_noktalar
WHERE ST_SRID(geom) = 4326;

-- İstatistikleri güncelle (sorgu planlayıcısı için)
ANALYZE noktalar_typed;

SELECT COUNT(*) AS aktarilan FROM noktalar_typed;`}
  label="Typed tabloya aktar + ANALYZE"
  description="GEOMETRY(POINT, 4326) tip kısıtlaması tabloya yazılabilecek geometri türünü ve koordinat sistemini sınırlar; bu da yanlış geometri tipinin eklenmesini önler. ANALYZE komutu tablonun istatistiklerini güncelleyerek sorgu planlayıcısının daha iyi indeks ve JOIN kararları almasını sağlar. Büyük INSERT veya UPDATE işlemleri sonrasında çalıştırılması önerilir."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- UTM noktalarını 4326'ya dönüştür
WITH utm_noktalar AS (
  SELECT 'A Noktası' AS ad, ST_SetSRID(ST_MakePoint(649000, 4542000), 32635) AS geom
  UNION ALL SELECT 'B Noktası', ST_SetSRID(ST_MakePoint(651000, 4540000), 32635)
  UNION ALL SELECT 'C Noktası', ST_SetSRID(ST_MakePoint(647000, 4544000), 32635)
)
SELECT
  ad,
  ST_AsText(geom)                    AS utm_35n,
  ST_AsText(ST_Transform(geom,4326)) AS wgs84,
  ST_Transform(geom,4326)            AS geom
FROM utm_noktalar;`}
  label="UTM → WGS84 toplu dönüşüm"
  autoFit
  description="UTM 35N (EPSG:32635) koordinatlarındaki üç nokta ST_Transform ile WGS84'e dönüştürülür. utm_35n sütunu orijinal metre cinsinden koordinatları, wgs84 sütunu dönüştürülmüş derece cinsinden koordinatları gösterir. geom sütunu harita bileşeni için WGS84 geometrisi döndürür. Bu dönüşüm tersine çevrilebilir: ST_Transform(geom_4326, 32635) ile geri UTM'e dönüştürülebilir."
/>

## Yaygın Hatalar

1. **UPDATE öncesinde yedek alınmalıdır.** `CREATE TABLE backup AS SELECT * FROM tablo` ile anlık kopya oluşturulmalıdır; `UPDATE` işlemi geri alınamaz.
2. **ANALYZE ne zaman gereklidir.** Tablo verisinin %10'undan fazlası değiştiğinde sorgu planlayıcı eski istatistiklerle çalışır. Büyük INSERT veya UPDATE işlemleri sonrasında `ANALYZE tablo` çalıştırılmalıdır.
3. **WHERE koşulsuz `ST_Transform`.** Zaten EPSG:4326'da olan satırlara WHERE koşulu eklenmezse `4326 → 4326` dönüşümü gereksiz yere her satır için tekrarlanır; işlem yavaşlar ancak veriyi bozmaz.
