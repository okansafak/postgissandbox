## 🎯 Ne öğreneceksin?

- `PARTITION BY LIST` sözdizimini yazabileceksin
- Kategorik sütunlara (ülke, şehir, tür) göre bölümleme yapabileceksin
- Default partition ile beklenmedik değerleri yönetebileceksin

---

## 📖 Kavram: List Partitioning

**Range partitioning:** Sürekli değer aralığı (tarih, sayı)
**List partitioning:** Ayrık kategoriler (ülke kodu, şehir, tür)

```sql
CREATE TABLE mekanlar_ulke (
  id     BIGSERIAL,
  ulke   CHAR(2) NOT NULL,
  ad     TEXT,
  geom   GEOMETRY(POINT, 4326)
) PARTITION BY LIST (ulke);

CREATE TABLE mekanlar_tr PARTITION OF mekanlar_ulke
  FOR VALUES IN ('TR');

CREATE TABLE mekanlar_gr PARTITION OF mekanlar_ulke
  FOR VALUES IN ('GR');

CREATE TABLE mekanlar_other PARTITION OF mekanlar_ulke
  DEFAULT;
```

## ▶ List Partition Pruning

## ⚠ Tuzaklar

1. **Default partition var mı?** Yoksa tanımsız değer `INSERT` hatası verir. Her zaman `DEFAULT` partition oluştur.
2. **List partition çok parçalanırsa:** 100 ülke için 100 partition — planner overhead'i artar. 10-20'den fazla partition için hash partitioning veya range düşün.
3. **Partition key güncelleme (UPDATE):** `UPDATE SET sehir = 'ankara' WHERE sehir = 'istanbul'` — satır eski partition'dan silinir, yenisine eklenir. PostgreSQL 10 öncesinde bu desteklenmiyordu.
