## Öğrenim Hedefleri

Range partitioning sürekli değer aralıkları için uygundur; ancak ülke kodu, şehir adı veya kategori gibi ayrık değerlerin yönetimi için List Partitioning daha doğal bir modelleme sağlar. Bu ders, kategorik sütunlara göre bölümleme yöntemini ele almaktadır.

- `PARTITION BY LIST` sözdizimini yazmak
- Kategorik sütunlara göre bölümleme yapmak
- Default partition ile beklenmedik değerleri yönetmek

---

## List Partitioning

**Range partitioning:** Sürekli değer aralığı (tarih, sayı)

**List partitioning:** Ayrık kategoriler (ülke kodu, şehir, tür)

```sql
CREATE TABLE mekanlar_ulke (
  id     BIGSERIAL,
  ulke   CHAR(2) NOT NULL,
  ad     TEXT,
  geom   GEOMETRY(POINT, 4326)
) PARTITION BY LIST (ulke);

CREATE TABLE mekanlar_tr PARTITION OF mekanlar_ulke
  FOR VALUES IN ('TR');

CREATE TABLE mekanlar_gr PARTITION OF mekanlar_ulke
  FOR VALUES IN ('GR');

-- Tanımsız değerler için zorunlu
CREATE TABLE mekanlar_other PARTITION OF mekanlar_ulke
  DEFAULT;
```

### List partition pruning

<RunnableBlock
  sql={`-- Sadece istanbul bölümü taranır
EXPLAIN
SELECT * FROM pois_sehir
WHERE sehir = 'istanbul'
  AND tur = 'restoran';`}
  label="List partition pruning"
  description="WHERE sehir = 'istanbul' koşuluyla yapılan sorguda planlayıcı yalnızca pois_istanbul bölümünü tarar; diğer bölümler (pois_ankara, pois_izmir, pois_diger) planda yer almaz. tur = 'restoran' koşulu partition key olmadığından pruning yapmaz; yalnızca istanbul bölümünde filtre olarak uygulanır."

## Pratik Uygulama

<RunnableBlock
  sql={`-- 'bursa' default'a düştü
SELECT tableoid::regclass AS partition, COUNT(*)
FROM pois_sehir
GROUP BY 1;

-- Default'taki kayıtlar
SELECT * FROM pois_diger;`}
  label="Default partition içeriği"
  description="Şehir başına kayıt sayısı ve her kaydın hangi bölümde bulunduğu listelenir. 'bursa' için özel bölüm tanımlanmadığından bu kayıt pois_diger (DEFAULT) bölümüne düşer. SELECT * FROM pois_diger bu kaydı gösterir. Yeni şehirler eklendikçe DEFAULT bölümü büyüyebilir; büyüyen kategoriler için yeni bölüm oluşturulup kayıtlar taşınabilir."

## Yaygın Hatalar

1. **Default partition yoksa tanımsız değer hata verir.** Partition listesinde bulunmayan bir değer için `INSERT` hatası oluşur. Her list partition tablosuna mutlaka `DEFAULT` bölümü eklenmelidir.
2. **Çok fazla bölüm planlayıcı overhead'ini artırır.** 100 ülke için 100 bölüm tanımlamak planlayıcının tüm bölümleri değerlendirmesi için ek maliyet getirir. 10–20'den fazla bölüm için hash partitioning veya farklı bir strateji düşünülmelidir.
3. **Partition key güncellemesi mümkün ancak maliyetlidir.** `UPDATE SET sehir = 'ankara' WHERE sehir = 'istanbul'` satırı eski bölümden siler ve yeni bölüme ekler; bu işlem iki fiziksel I/O olayı gerektirir.
