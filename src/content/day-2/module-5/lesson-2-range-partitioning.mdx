## Öğrenim Hedefleri

Milyonlarca kayıt içeren büyük tablolar belirli bir noktadan sonra düz indekslerle yönetilemez hale gelir. Tablo bölümleme (partitioning), veriyi mantıksal parçalara ayırarak hem sorgu performansını hem de bakım verimliliğini artırır. Bu ders, tarih veya sayısal aralığa göre bölümlemeyi ele almaktadır.

- PostgreSQL declarative partitioning sözdizimini yazmak
- `PARTITION BY RANGE` ile zaman veya sayısal bölümleme yapmak
- Partition pruning'in nasıl çalıştığını EXPLAIN ile doğrulamak

---

## Declarative Partitioning

Büyük tablolarda sorgu performansı düşer; çözüm veriyi mantıksal parçalara bölmektir.

**Range Partitioning:** Belirli sütunun değer aralığına göre bölümleme.

```sql
-- Ana tablo (partition parent)
CREATE TABLE olcumler (
  id         BIGSERIAL,
  zaman      TIMESTAMPTZ NOT NULL,
  geom       GEOMETRY(POINT, 4326),
  sicaklik   REAL
) PARTITION BY RANGE (zaman);

-- Bölüm (partition)
CREATE TABLE olcumler_2024_q1
  PARTITION OF olcumler
  FOR VALUES FROM ('2024-01-01') TO ('2024-04-01');
```

**Avantaj:** `WHERE zaman BETWEEN '2024-01' AND '2024-04'` → yalnızca ilgili bölüm okunur (**partition pruning**).

CREATE TABLE IF NOT EXISTS gps_kayit (
  id     BIGSERIAL,
  arac   TEXT,
  zaman  TIMESTAMPTZ NOT NULL,
  geom   GEOMETRY(POINT, 4326)
) PARTITION BY RANGE (zaman);

-- Aylık bölümler
CREATE TABLE IF NOT EXISTS gps_kayit_2024_01
  PARTITION OF gps_kayit
  FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');

CREATE TABLE IF NOT EXISTS gps_kayit_2024_02
  PARTITION OF gps_kayit
  FOR VALUES FROM ('2024-02-01') TO ('2024-03-01');

CREATE TABLE IF NOT EXISTS gps_kayit_2024_03
  PARTITION OF gps_kayit
  FOR VALUES FROM ('2024-03-01') TO ('2024-04-01');

-- Veri ekle
INSERT INTO gps_kayit (arac, zaman, geom)
SELECT
  'Araç-' || (1 + i % 5),
  '2024-01-01'::timestamptz + (i || ' hours')::interval,
  ST_SetSRID(ST_MakePoint(28.9 + random()*0.5, 40.9 + random()*0.3), 4326)
FROM generate_series(1, 1500) i;

SELECT tableoid::regclass AS partition, COUNT(*) AS kayit
FROM gps_kayit
GROUP BY tableoid;`}

### Partition pruning gözlemi

<RunnableBlock
  sql={`-- Partition pruning: sadece Ocak bölümü taranır
EXPLAIN
SELECT COUNT(*) FROM gps_kayit
WHERE zaman BETWEEN '2024-01-01' AND '2024-01-31 23:59:59';`}
  label="Partition pruning — EXPLAIN"
  description="Yalnızca Ocak 2024 tarih aralığını kapsayan sorgunun planı incelenir. EXPLAIN çıktısında gps_kayit_2024_02 ve gps_kayit_2024_03 bölümlerinin planda yer almaması pruning'in çalıştığını gösterir. Pruning olmadan üç bölüm de taranacaktır; pruning ile yalnızca ilgili bölüm okunur."
/>

### Partition üzerine GiST indeksi

<RunnableBlock
  sql={`-- Her bölüme ayrı GiST indeks
CREATE INDEX IF NOT EXISTS idx_gps_kayit_01_geom
  ON gps_kayit_2024_01 USING gist(geom);

CREATE INDEX IF NOT EXISTS idx_gps_kayit_02_geom
  ON gps_kayit_2024_02 USING gist(geom);

CREATE INDEX IF NOT EXISTS idx_gps_kayit_03_geom
  ON gps_kayit_2024_03 USING gist(geom);

ANALYZE gps_kayit;

-- Hem zaman hem mekan filtresi → tek bölüm + indeks
EXPLAIN ANALYZE
SELECT COUNT(*) FROM gps_kayit
WHERE zaman >= '2024-01-01' AND zaman < '2024-02-01'
  AND ST_DWithin(
    geom::geography,
    ST_SetSRID(ST_MakePoint(29.1, 41.0),4326)::geography,
    20000);`}
  label="Partition + GiST indeks birlikte"
  description="Her bölüme ayrı GiST indeksi eklendikten sonra hem zaman hem mekan filtresi içeren sorgu incelenir. Planlayıcı önce pruning ile Ocak 2024 bölümünü seçer, ardından bu bölümdeki GiST indeksini kullanarak 20 km içindeki noktaları bulur. EXPLAIN ANALYZE çıktısında tek bölüm adı ve Index Scan beklenir."

## Pratik Uygulama

<RunnableBlock
  sql={`-- Hangi bölümler var, boyutları ne?
SELECT
  child.relname AS partition,
  pg_size_pretty(pg_relation_size(child.oid)) AS boyut,
  (SELECT COUNT(*) FROM gps_kayit
   WHERE tableoid = child.oid) AS satir
FROM pg_inherits
JOIN pg_class parent ON pg_inherits.inhparent = parent.oid
JOIN pg_class child  ON pg_inherits.inhrelid  = child.oid
WHERE parent.relname = 'gps_kayit';`}
  label="Partition listesi ve boyutları"
  description="pg_inherits kataloğu bölümlü tablo ile fiziksel bölümler arasındaki ilişkiyi tutar. Bu sorgu her bölümün adını, disk boyutunu ve satır sayısını listeler. Boyut dengesi bölüm stratejisinin ne kadar verimli olduğunu gösterir; aşırı dengesiz bölümler yeniden tasarım gerektirebilir."
/>

## Yaygın Hatalar

1. **Partition key NULL değer kabul etmez.** `PARTITION BY RANGE (zaman)` tanımlı tabloya `zaman IS NULL` değeriyle satır eklendiğinde hata oluşur. `CREATE TABLE ... DEFAULT` bölümü oluşturarak bu durum yönetilebilir.
2. **Üst tabloya eklenen indeks yeni bölümleri kapsamaz.** Her yeni bölüm için ayrı indeks oluşturulmalı ya da `pg_partman` extension ile otomatikleştirilmelidir.
3. **Çok küçük bölümler overhead artırır.** Saatlik bölümleme çoğu senaryo için gereksizdir; aylık veya üç aylık bölümleme genellikle yeterlidir.
