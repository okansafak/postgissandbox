## Öğrenim Hedefleri

GiST indeksi oluşturma sözdizimini bilmek yeterli değildir; üretim ortamında hangi indeks varyantının ne zaman kullanılacağını da bilmek gerekir. Bu ders, temel, koşullu ve cast'lı sütun için GiST indeksi oluşturma yöntemlerini ele almaktadır.

- `CREATE INDEX USING gist` sözdizimini yazmak
- Concurrent indeks oluşturmanın farkını açıklamak
- `pg_indexes` ile mevcut indeksleri sorgulamak

---

## CREATE INDEX Sözdizimi

```sql
-- Temel
CREATE INDEX idx_ad ON tablo USING gist(geom);

-- Concurrent (tablo kilitlemez, canlı sistemde önerilir)
CREATE INDEX CONCURRENTLY idx_ad ON tablo USING gist(geom);

-- Partial index (koşullu — daha küçük, daha hızlı)
CREATE INDEX idx_aktif ON tablo USING gist(geom) WHERE aktif = TRUE;

-- Functional index (cast'lı sütun)
CREATE INDEX idx_geog ON tablo USING gist(geography(geom));
```

WHERE tablename = 'mekanlar_idx';`}

<RunnableBlock

## Yaygın Hatalar

1. **`CONCURRENTLY` transaction bloğu içinde çalışmaz.** `CREATE INDEX CONCURRENTLY` bir `BEGIN...COMMIT` bloğunun içine alınamaz; bağımsız bir ifade olarak çalıştırılmalıdır.
2. **Partial index WHERE sorguyla birebir eşleşmelidir.** `WHERE aktif = TRUE` koşuluyla oluşturulan indeks, `WHERE aktif = FALSE` sorgusunda kullanılmaz; PostgreSQL koşul eşleşmesini katı biçimde kontrol eder.
3. **Gereksiz indeksler yazma performansını düşürür.** Her INSERT ve UPDATE tüm indeksleri günceller. `pg_stat_user_indexes.idx_scan = 0` olan indeksler kaldırılmalıdır.
