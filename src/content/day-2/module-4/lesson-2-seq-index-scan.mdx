## Öğrenim Hedefleri

İndeks var diye her sorguda index scan kullanılmaz; planlayıcı her zaman en ucuz yolu tercih eder. Bu derste üç temel scan tipinin ne anlama geldiği, hangi koşullarda hangi scan'ın seçildiği ve planlayıcı kararlarının nasıl test edilebileceği ele alınmaktadır.

- Sequential Scan, Index Scan ve Bitmap Heap Scan arasındaki farkı açıklamak
- Planlayıcının indeks varken Seq Scan seçtiği durumları anlamak
- `enable_seqscan` ile zorlamalı test yapmak

---

## Üç Temel Scan Tipi

| Scan Tipi | Ne zaman seçilir? | İndeks kullanır mı? |
|---|---|---|
| **Seq Scan** | Tablonun büyük bölümü okunacak | Hayır |
| **Index Scan** | Az satır döndürecek, seçici sorgu | Evet |
| **Bitmap Heap Scan** | Orta seçicilik; önce indeks, sonra heap | Evet (Bitmap Index Scan ile) |

**Önemli:** Planlayıcı maliyet hesabı yapar:
- 1000 satırdan 900'ü okunacaksa → Seq Scan daha ucuz (sıralı I/O)
- 1000 satırdan 10'u okunacaksa → Index Scan daha ucuz (seçici erişim)

## Uygulama Örneği

### Büyük seçim — Seq Scan

<RunnableBlock
  sql={`-- Hazırlık tablosu (5000 kayıt)
CREATE TABLE IF NOT EXISTS noktalar_scan (
  id   SERIAL PRIMARY KEY,
  tur  TEXT,
  geom GEOMETRY(POINT, 4326)
);

INSERT INTO noktalar_scan (tur, geom)
SELECT
  CASE WHEN i % 100 = 0 THEN 'ozel' ELSE 'normal' END,
  ST_SetSRID(ST_MakePoint(26+random()*19, 36+random()*6), 4326)
FROM generate_series(1, 5000) i
ON CONFLICT DO NOTHING;

CREATE INDEX IF NOT EXISTS idx_scan_geom ON noktalar_scan USING gist(geom);
ANALYZE noktalar_scan;

-- Büyük seçim → Seq Scan beklenir
EXPLAIN SELECT COUNT(*) FROM noktalar_scan WHERE tur = 'normal';`}
  label="Büyük seçim — Seq Scan"
  description="5000 kaydın 4950'si (yüzde 99) 'normal' türündedir. Bu kadar büyük bir seçimde indeks kullanmak (rastgele erişim) sıralı taramadan daha pahalıdır; planlayıcı Seq Scan seçer. tur sütununda GiST indeksi yoktur; ancak geom üzerindeki GiST indeksi de bu sorgu için uygulanamaz. EXPLAIN çıktısında 'Seq Scan on noktalar_scan' beklenir."
/>

### Küçük alan — Index Scan

<RunnableBlock
  sql={`-- Küçük alan sorgusu → Index Scan beklenir
EXPLAIN ANALYZE
SELECT * FROM noktalar_scan
WHERE ST_DWithin(
  geom::geography,
  ST_SetSRID(ST_MakePoint(28.979,41.015),4326)::geography,
  10000  -- 10 km
);`}
  label="Küçük alan — Index Scan"
  description="10 km yarıçaplı dar bir alan sorgusu tablonun çok küçük bir bölümünü döndürür. Bu koşulda GiST indeks taraması Seq Scan'dan çok daha ucuzdur; planlayıcı 'Index Scan' veya 'Bitmap Index Scan' seçer. 'actual rows' gerçekte dönen satır sayısını gösterir; bu değer 5000'den çok küçük olmalıdır."
/>

### Geniş alan — Bitmap Heap Scan

<RunnableBlock
  sql={`-- Orta boyut alan → Bitmap Heap Scan
EXPLAIN ANALYZE
SELECT * FROM noktalar_scan
WHERE ST_DWithin(
  geom::geography,
  ST_SetSRID(ST_MakePoint(28.979,41.015),4326)::geography,
  500000  -- 500 km — geniş alan
);`}
  label="Geniş alan — Bitmap Heap Scan"
  description="500 km yarıçaplı geniş alan Türkiye'nin önemli bir bölümünü kapsar; tablonun orta büyüklükte bir kısmı döner. Bu durumda planlayıcı Bitmap Heap Scan seçebilir: önce 'Bitmap Index Scan' ile GiST indeksini kullanarak aday sayfalara bir bit haritası oluşturur, ardından 'Bitmap Heap Scan' ile bu sayfalardaki satırları okur. Bu yaklaşım tam Index Scan'dan ve Seq Scan'dan daha verimli olabilir."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Seq scan'ı geçici kapat — planner index'e zorla
SET enable_seqscan = off;

EXPLAIN ANALYZE
SELECT COUNT(*) FROM noktalar_scan WHERE tur = 'normal';

-- Geri aç!
SET enable_seqscan = on;`}
  label="Seq Scan kapatılınca plan değişir"
  description="enable_seqscan = off planlayıcıyı Seq Scan'ı yüksek maliyetli göstermeye zorlar; alternatif varsa Index Scan seçilir. Bir INDEX SCAN seçildiğinde maliyet neden Seq Scan'dan yüksektir — bu, planlayıcının doğru seçim yaptığını doğrular. Bu ayar yalnızca test amaçlıdır; üretim ortamında değiştirilmemelidir."
/>

## Yaygın Hatalar

1. **Planlayıcı her zaman doğru değildir.** İstatistikler eskiyse yanlış plan seçilebilir. `ANALYZE` çalıştırılmalıdır.
2. **Küçük tablolarda Seq Scan normaldir.** 100 satırlık bir tablo için Index Scan ek yükü (random I/O, indeks okuma) Seq Scan'dan pahalıdır; bu planlayıcı hatası değildir.
3. **Bitmap Scan `Recheck Cond` içerir.** EXPLAIN çıktısında `Recheck Cond` satırı, indeks taraması sonrasında gerçek geometri testinin yapıldığını gösterir; bu iki aşamalı filtrelemenin kesin faz adımıdır.
