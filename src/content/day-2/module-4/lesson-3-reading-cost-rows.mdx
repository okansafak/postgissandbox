## 🎯 Ne öğreneceksin?

- `cost=X..Y` değerini yorumlayabileceksin
- Tahmini rows ile gerçek rows farkını tespit edebileceksin
- Kötü tahminin nasıl düzeltileceğini söyleyebileceksin

---

## 📖 Kavram: Cost Modeli

PostgreSQL planlayıcısı "maliyeti" şu referanslara göre hesaplar:

| Parametre | Varsayılan | Anlamı |
|---|---|---|
| `seq_page_cost` | 1.0 | Sıralı disk okuma birimi |
| `random_page_cost` | 4.0 | Rastgele disk okuma (index jump) |
| `cpu_tuple_cost` | 0.01 | Satır işleme |
| `cpu_operator_cost` | 0.0025 | Operatör değerlendirme |

**Örnek:** 10.000 satır seq scan = `10000 × 0.01 = 100` CPU cost + disk page cost.

**Önemli:** Cost sayısı mutlak değil — sadece karşılaştırma içindir. `cost=50` iyi mi kötü mü? Tek başına anlamı yok; alternatiflerle karşılaştırılır.

---

## 📖 Rows Tahmini Doğruluğu

```
(cost=0.00..45.00 rows=50 width=32)
  (actual time=0.1..8.2 rows=3421 loops=1)
              ↑              ↑
           tahmin          gerçek
           50 satır       3421 satır → %6844 hata!
```

Bu kadar büyük fark varsa:
1. `ANALYZE` çalıştırılmamış
2. Korelasyon yüksek (spatial clustering)
3. `default_statistics_target` çok düşük

## ▶ Canlı Örnek — Tahmin Hatası Tespit

<RunnableBlock
  sql={`CREATE TABLE IF NOT EXISTS skewed_points (
  id   SERIAL PRIMARY KEY,
  bolge TEXT,
  geom GEOMETRY(POINT, 4326)
);

-- İstanbul bölgesine yoğunlaştır (%80)
INSERT INTO skewed_points (bolge, geom)
SELECT
  'istanbul',
  ST_SetSRID(ST_MakePoint(28.5 + random()*1.5, 40.8 + random()*0.8), 4326)
FROM generate_series(1, 4000);

-- Diğer bölgeler (%20)
INSERT INTO skewed_points (bolge, geom)
SELECT
  'diger',
  ST_SetSRID(ST_MakePoint(26+random()*19, 36+random()*6), 4326)
FROM generate_series(1, 1000);

-- ANALYZE çalıştırma — tahmin kötü olacak
EXPLAIN SELECT COUNT(*) FROM skewed_points
WHERE ST_DWithin(
  geom::geography,
  ST_SetSRID(ST_MakePoint(28.979,41.015),4326)::geography,
  100000);`}
  label="ANALYZE öncesi — kötü tahmin"
/>

## 🧪 Dene — Loops Değeri

<RunnableBlock
  sql={`-- Nested loop join: loops > 1 olduğunda gerçek satır = rows × loops
EXPLAIN ANALYZE
SELECT a.id, b.id
FROM skewed_points a
JOIN skewed_points b
  ON ST_DWithin(a.geom::geography, b.geom::geography, 5000)
WHERE a.bolge = 'istanbul'
LIMIT 20;`}
  label="Nested loop — loops değeri"
/>

## ⚠ Tuzaklar

1. **rows tahmini `loops` ile çarpılmalı:** `actual rows=5 loops=1000` → toplam 5000 satır işlendi. Rapor bunu göstermez — elle çarp.
2. **cost sadece I/O değil:** CPU ve bellek de dahil. Yüksek cost ≠ yavaş disk.
3. **Tahmin hatasını tolere et:** %50 hata çoğu zaman plan kararını değiştirmez. %1000+ hata ciddi sorun.
