## Öğrenim Hedefleri

- Row Level Security (RLS) kavramını açıklamak
- `CREATE POLICY` ile satır filtresi yazmak
- `current_user` ve `current_setting` ile dinamik politika oluşturmak

---

## Row Level Security

Normal yetki modeli tablo düzeyinde çalışır: tüm tablo okunabilir veya hiçbiri.
**RLS:** Her kullanıcı yalnızca kendi satırlarını görür.

```sql
-- 1. RLS etkinleştir
ALTER TABLE t ENABLE ROW LEVEL SECURITY;

-- 2. Politika yaz
CREATE POLICY p ON t FOR SELECT
  USING (sahip = current_user);
```

Politika tanımlı olmayan bir tabloda RLS aktifse hiçbir satır döner (deny-by-default).

### Session değişkeni ile RLS

<RunnableBlock
  sql={`-- Session parametresiyle dinamik filtre
CREATE TABLE IF NOT EXISTS bolge_verileri (
  id     SERIAL PRIMARY KEY,
  il_kod TEXT,
  ad     TEXT,
  geom   GEOMETRY(POINT, 4326)
);

INSERT INTO bolge_verileri (il_kod, ad, geom) VALUES
  ('34', 'İstanbul Noktası', ST_SetSRID(ST_MakePoint(28.979, 41.037), 4326)),
  ('06', 'Ankara Noktası',   ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326)),
  ('35', 'İzmir Noktası',    ST_SetSRID(ST_MakePoint(27.142, 38.423), 4326));

ALTER TABLE bolge_verileri ENABLE ROW LEVEL SECURITY;

-- Politika: session'da ayarlanan il koduna göre filtrele
CREATE POLICY il_politika ON bolge_verileri
  FOR SELECT
  USING (
    il_kod = current_setting('app.il_kodu', TRUE)
    OR current_setting('app.il_kodu', TRUE) IS NULL
  );

-- Test: İstanbul için filtrele
SET app.il_kodu = '34';
SELECT * FROM bolge_verileri;`}
  label="Session değişkeni ile RLS"
  description="current_setting('app.il_kodu', TRUE) oturum değişkenini okur; TRUE parametresi değişken tanımlı değilse NULL döndürür (hata vermez). SET app.il_kodu = '34' oturuma özel parametre ayarlar; politika bu değere göre satırları filtreler. OR IS NULL koşulu parametresiz oturumlarda tüm satırları gösterir."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Ankara için filtrele
SET app.il_kodu = '06';
SELECT il_kod, ad FROM bolge_verileri;`}
  label="Ankara il filtresi"

<RunnableBlock

SET app.il_kodu = '';
SELECT il_kod, ad FROM bolge_verileri;`}
  label="Filtre kaldır"
  description="Boş string atanması current_setting('app.il_kodu', TRUE) değerini boş string yapar. Politikadaki il_kod = '' koşulu hiçbir il kodu ile eşleşmez; OR IS NULL da sağlanmaz. Bu senaryoda tüm satırlar gizlenir. Gerçek 'tümünü gör' davranışı için SET app.il_kodu = NULL; veya RESET app.il_kodu; kullanılmalıdır."

## Yaygın Hatalar

1. **Superuser RLS'yi atlar:** `BYPASSRLS` yetkisine sahip kullanıcılar politikayı görmez. Test için normal kullanıcıya geçilmelidir.
2. **`current_setting` ikinci parametre:** `current_setting('app.x', TRUE)` — TRUE ile ayar yoksa NULL döner; FALSE ile hata üretir. TRUE her zaman tercih edilmelidir.
3. **Performans:** RLS politikası her sorguya WHERE ekler. Politika sütununda indeks olmazsa performans düşer; `il_kod` üzerine B-tree indeks eklenmelidir.
