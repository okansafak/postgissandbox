## 🎯 Ne öğreneceksin?

- `ST_Within` ile mekansal RLS politikası yazabileceksin
- Kullanıcı yetki alanı poligonu bazlı erişim kısıtlaması yapabileceksin
- Mekansal politikanın performans etkisini değerlendirebileceksin

---

## 📖 Kavram: Mekansal Erişim Kontrolü

"İstanbul sorumlusu sadece İstanbul içindeki parselleri görsün."

```sql
CREATE POLICY mekansal_sinir ON parseller
  FOR SELECT
  USING (
    ST_Within(
      geom,
      (SELECT bolge FROM kullanici_bolgeleri WHERE kullanici = current_user)
    )
  );
```

## ▶ Mekansal RLS Politikası

<RunnableBlock
  sql={`ALTER TABLE yetkili_noktalar ENABLE ROW LEVEL SECURITY;

-- Mekansal politika: kullanıcının yetki alanı içindeki noktalar
CREATE POLICY mekansal_erisim ON yetkili_noktalar
  FOR SELECT
  USING (
    ST_Within(
      geom,
      COALESCE(
        (SELECT bolge FROM kullanici_bolge WHERE kullanici = current_user),
        ST_MakeEnvelope(-180, -90, 180, 90, 4326)  -- superuser için tüm dünya
      )
    )
  );

-- Superuser her şeyi görür (politika bypass)
SELECT ad, ST_AsText(geom) AS konum FROM yetkili_noktalar;`}
  label="Mekansal RLS politikası"
/>

## ▶ Session Parametresi ile Simüle Et

<RunnableBlock
  sql={`-- Politika aktif ama superuser — manuel filtre ile test
-- Gerçek sistemde: SET ROLE istanbul_usr; ile test edilir

  n.ad,

## 🧪 Dene — Ankara Kullanıcısı

<RunnableBlock
  sql={`-- Ankara kullanıcısının görebileceği noktalar
SELECT
  n.ad,
  ROUND(ST_Distance(n.geom::geography, ST_Centroid(b.bolge)::geography)::numeric / 1000, 1) AS merkeze_km
FROM yetkili_noktalar n
JOIN kullanici_bolge b ON b.kullanici = 'ankara_usr'
WHERE ST_Within(n.geom, b.bolge);`}

## ⚠ Tuzaklar

1. **Mekansal politika GIST indeksi kullanmaz doğrudan:** `ST_Within(geom, subquery)` her satırda subquery çalıştırır. Performans için: yetki alanını oturumda önbelleğe al veya materialized view kullan.
2. **`current_user` vs `session_user`:** Rol değiştirildiğinde `current_user` değişir, `session_user` değişmez. Politikada `current_user` güvenli.
3. **NULL geometri:** Kullanıcının yetki alanı yoksa `COALESCE` ile fallback ver — aksi halde politika her satırı reddeder.
