## Öğrenim Hedefleri

- GeoServer ve QGIS için güvenli read-only kullanıcı oluşturmak
- Şema bazlı yetki modelini yapılandırmak
- Üretim ortamı güvenlik şablonunu açıklamak

---

## Read-Only Üretim Kullanıcısı

CBS uygulamaları (GeoServer, QGIS, Grafana) genellikle yalnızca SELECT yetkisi gerektirir. Yanlış yapılandırmada INSERT/UPDATE yapılabilmesi veri bütünlüğü açısından ciddi risk oluşturur.

**En az yetki şablonu:**
```sql
CREATE ROLE geoserver_ro NOLOGIN;
GRANT CONNECT ON DATABASE cbs TO geoserver_ro;
GRANT USAGE ON SCHEMA public TO geoserver_ro;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO geoserver_ro;
ALTER DEFAULT PRIVILEGES IN SCHEMA public
  GRANT SELECT ON TABLES TO geoserver_ro;

CREATE ROLE geoserver LOGIN PASSWORD 'güçlü_şifre';
GRANT geoserver_ro TO geoserver;
```

## Uygulama Örneği

### Read-only yetki şablonu

<RunnableBlock
  sql={`-- Read-only grup rolü
CREATE ROLE IF NOT EXISTS ro_kullanici NOLOGIN;

-- Mevcut tüm tablolar için SELECT
GRANT USAGE ON SCHEMA public TO ro_kullanici;

-- Tek tek tablo yetkisi
GRANT SELECT ON noktalar_perf TO ro_kullanici;
GRANT SELECT ON kaza_noktalari TO ro_kullanici;
GRANT SELECT ON bolge_verileri TO ro_kullanici;

-- Yetki kontrolü
SELECT
  table_name,
  privilege_type
FROM information_schema.role_table_grants
WHERE grantee = 'ro_kullanici'
ORDER BY table_name;`}
  label="Read-only yetki şablonu"
  description="ro_kullanici NOLOGIN grup rolü üretimde GeoServer veya QGIS gibi uygulamaların kullanıcılarının üye olacağı şablondur. GRANT USAGE ON SCHEMA schema içindeki nesnelere erişim kapısını açar. Her tablo için ayrı SELECT yetkisi least privilege prensibini uygular. information_schema.role_table_grants verilen yetkilerin doğru kaydedildiğini doğrular."
/>

### Fonksiyon yetkileri

<RunnableBlock
  sql={`-- Stored function için EXECUTE yetkisi
-- (PostGIS fonksiyonları public şemada)
GRANT EXECUTE ON FUNCTION ST_AsGeoJSON(geometry) TO ro_kullanici;
GRANT EXECUTE ON FUNCTION ST_Distance(geography, geography) TO ro_kullanici;

-- Daha genel: tüm public fonksiyonlar
-- GRANT EXECUTE ON ALL FUNCTIONS IN SCHEMA public TO ro_kullanici;

-- Verilen execute yetkileri
SELECT
  routine_name,
  privilege_type
FROM information_schema.role_routine_grants
WHERE grantee = 'ro_kullanici'
  AND routine_schema = 'public'
LIMIT 10;`}
  label="Fonksiyon yetkileri"
  description="Tablo SELECT yetkisi PostGIS fonksiyonlarını kapsamaz; ST_AsGeoJSON ve ST_Distance gibi sık kullanılan fonksiyonlar için ayrıca EXECUTE yetkisi verilmesi gerekir. GRANT EXECUTE ON ALL FUNCTIONS daha kapsayıcı ama daha az kısıtlayıcıdır. information_schema.role_routine_grants verilen fonksiyon yetkilerini listeler."
/>

### Güvenlik denetim raporu

<RunnableBlock
  sql={`-- Hangi tablolara kim ne yetkiyle erişebilir?
SELECT
  grantee,
  table_schema,
  table_name,
  string_agg(privilege_type, ', ' ORDER BY privilege_type) AS yetkiler
FROM information_schema.role_table_grants
WHERE table_schema = 'public'
  AND grantee NOT IN ('postgres', 'PUBLIC', 'pg_monitor')
  AND grantee NOT LIKE 'pg_%'
GROUP BY grantee, table_schema, table_name
ORDER BY grantee, table_name;`}
  label="Güvenlik denetim raporu"
  description="information_schema.role_table_grants tüm nesne düzeyindeki yetkileri listeler. string_agg birden fazla yetkiyi tek satırda birleştirir. Sistem rolleri (postgres, PUBLIC, pg_%) ve pg_monitor filtrelenerek yalnızca uygulama rolleri gösterilir. Bu sorgu periyodik güvenlik denetimi için kullanılır."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Tüm özel rollerin yetki matrisi
SELECT
  r.rolname AS rol,
  r.rolcanlogin AS login,
  string_agg(DISTINCT m.rolname, ', ') AS uye_oldugu_roller
FROM pg_roles r
LEFT JOIN pg_auth_members am ON am.member = r.oid
LEFT JOIN pg_roles m ON m.oid = am.roleid
WHERE r.rolname NOT LIKE 'pg_%'
  AND r.rolname != 'postgres'
GROUP BY r.rolname, r.rolcanlogin
ORDER BY r.rolname;`}
  label="Rol üyelik matrisi"
  description="pg_auth_members rol üyeliklerini tutar. LEFT JOIN ile her rolün üye olduğu grup roller listelenir. string_agg birden fazla üyeliği virgülle birleştirir. rolcanlogin login kullanıcıları (TRUE) grup rollerinden (FALSE) ayırır. Bu matris rol hiyerarşisini ve yetki miras zincirini görselleştirir."
/>

## Yaygın Hatalar

1. **`ALTER DEFAULT PRIVILEGES`:** Mevcut tablolara `GRANT SELECT ON ALL TABLES` uygulanır; sonradan eklenen tablolar için `ALTER DEFAULT PRIVILEGES` zorunludur.
2. **pg_hba.conf PGlite'ta yok:** Gerçek sistemde bağlantı güvenliği `pg_hba.conf` ile yönetilir. PGlite'ta bu katman mevcut değildir.
3. **Şifre politikası:** `pg_password_check` extension ile güçlü şifre zorlama yapılabilir. Varsayılan PostgreSQL şifre karmaşıklığı kontrolü yapmaz.
