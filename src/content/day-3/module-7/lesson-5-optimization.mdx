## Capstone Bölüm 3: Rota Optimizasyonu ve Talep Tahmini

**Hedef:** Araç-sipariş atama, hareketli ortalama ile talep tahmini ve anomali tespiti uygulamak.

---

## Öğrenim Hedefleri

- Greedy insertion ile araç-sipariş atama algoritması uygulamak
- Hareketli ortalama ile saatlik talep tahmini üretmek
- Baseline karşılaştırması ile anomali tespiti yapmak

---

## Uygulama Örneği

### Araç-sipariş atama (greedy insertion)

<RunnableBlock
  sql={`-- Araç ve sipariş tabloları
CREATE TABLE IF NOT EXISTS orders (
  id INT PRIMARY KEY,
  location GEOMETRY(POINT, 4326),
  time_window_start TIME,
  time_window_end TIME,
  weight_kg NUMERIC
);

CREATE TABLE IF NOT EXISTS vehicles (
  id INT PRIMARY KEY,
  current_location GEOMETRY(POINT, 4326),
  capacity_kg INT,
  current_load_kg INT
);

-- Örnek veriler
INSERT INTO orders VALUES
  (1, ST_MakePoint(28.97, 41.00), '09:00', '12:00', 5),
  (2, ST_MakePoint(28.98, 41.01), '10:00', '14:00', 3),
  (3, ST_MakePoint(28.96, 41.02), '11:00', '15:00', 4)
ON CONFLICT DO NOTHING;

INSERT INTO vehicles VALUES
  (1, ST_MakePoint(28.95, 40.99), 50, 10),
  (2, ST_MakePoint(28.99, 41.03), 50, 15)
ON CONFLICT DO NOTHING;

-- Her sipariş için kapasite uygunluğu olan en yakın araç
WITH feasible AS (
  SELECT 
    o.id AS siparis_id,
    v.id AS arac_id,
    ST_Distance(v.current_location::geography, o.location::geography)::int4 AS mesafe_m
  FROM orders o, vehicles v
  WHERE v.current_load_kg + o.weight_kg <= v.capacity_kg
),
sirali AS (
  SELECT 
    siparis_id,
    arac_id,
    mesafe_m,
    ROW_NUMBER() OVER (PARTITION BY siparis_id ORDER BY mesafe_m) AS siralama
  FROM feasible
)
SELECT siparis_id, arac_id, mesafe_m
FROM sirali
WHERE siralama = 1
ORDER BY siparis_id;`}
  label="Araç-sipariş atama (greedy)"
  description="Greedy insertion her siparişi kapasite kısıtını karşılayan en yakın araca atar. WHERE v.current_load_kg + o.weight_kg <= v.capacity_kg kapasite filtresi uygunsuz araçları dışlar. ROW_NUMBER() OVER (PARTITION BY siparis_id ORDER BY mesafe_m) her sipariş için araçları mesafeye göre sıralar. WHERE siralama = 1 yalnızca en yakın aracı seçer; bu yaklaşım pencere fonksiyonunu CTE'ye taşıyarak WHERE filtresini geçerli kılar."
/>

### Saatlik talep tahmini

<RunnableBlock
  sql={`-- Günlük teslimat geçmişi
CREATE TABLE IF NOT EXISTS daily_deliveries (
  tarih DATE,
  saat INT,
  teslimat_sayisi INT
);

INSERT INTO daily_deliveries VALUES
  ('2024-01-01', 9, 45),
  ('2024-01-01', 10, 67),
  ('2024-01-01', 11, 89),
  ('2024-01-01', 12, 120),
  ('2024-01-02', 9, 48),
  ('2024-01-02', 10, 70)
ON CONFLICT DO NOTHING;

-- Hareketli ortalama ile 7 günlük tahmin
WITH saatlik_ort AS (
  SELECT 
    saat,
    AVG(teslimat_sayisi)::int4 AS ort_teslimat,
    STDDEV(teslimat_sayisi)::numeric(6,2) AS std_sapma
  FROM daily_deliveries
  GROUP BY saat
)
SELECT 
  saat,
  ort_teslimat AS tahmin,
  ROUND((ort_teslimat * 1.15)::numeric)::int4 AS pct15_tampon_ile,
  CEILING(ort_teslimat::numeric / 5)::int4 AS gereken_arac
FROM saatlik_ort
ORDER BY saat;`}
  label="Saatlik talep tahmini"
  description="AVG ve STDDEV saatlik geçmiş verileri istatistiksel olarak özetler. pct15_tampon_ile = tahmin * 1.15 talep patlamalarına karşı %15 tampon ekler; bu değer operasyonel güvenlik marjını temsil eder. CEILING(ort_teslimat / 5) her saat için gereken minimum araç sayısını hesaplar (araç başına 5 teslimat kapasitesi varsayımıyla). Tahmin doğruluğu artıkça std_sapma değeri küçülür."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Anlık anomali tespiti (baseline karşılaştırması)
WITH saatlik_istatistik AS (
  SELECT 
    EXTRACT(HOUR FROM CURRENT_TIMESTAMP)::int4 AS saat,
    COUNT(*) AS anlık_sayi,
    (SELECT AVG(teslimat_sayisi)::int4
     FROM daily_deliveries 
     WHERE saat = EXTRACT(HOUR FROM CURRENT_TIMESTAMP)::int4
    ) AS beklenen_sayi
  FROM daily_deliveries
  WHERE tarih = CURRENT_DATE
  GROUP BY saat
)
SELECT 
  saat,
  anlık_sayi,
  beklenen_sayi,
  CASE 
    WHEN beklenen_sayi IS NULL     THEN 'BASELINE YOK'
    WHEN anlık_sayi > beklenen_sayi * 1.5 THEN 'ANOMALI - YUKSEK'
    WHEN anlık_sayi < beklenen_sayi * 0.5 THEN 'ANOMALI - DUSUK'
    ELSE 'NORMAL'
  END AS durum
FROM saatlik_istatistik;`}
  label="Anlık anomali tespiti"
  description="Anomali tespiti mevcut saatin değerini geçmiş aynı saatin ortalamasıyla karşılaştırır. Beklenen değerin 1.5 katını aşan sayı yüksek anomali, 0.5 katının altına düşen ise düşük anomali olarak sınıflandırılır. Bu eşikler ±%50 sapmaya karşılık gelir; gerçek uygulamada z-skoru veya IQR tabanlı adaptif eşikler tercih edilir. beklened_sayi IS NULL baseline verisi olmayan saatler için uyarı verir."
/>
