## Capstone Bölüm 2: ETL ve Veri Entegrasyonu

**Hedef:** Farklı kaynaklardan gelen adres ve konum verilerini dönüştürmek, doğrulamak ve artımlı yükleme stratejisiyle güncel tutmak.

---

## Öğrenim Hedefleri

- Staging tablosu ve ETL pipeline mimarisi kurmak
- UTM → WGS84 koordinat dönüşümü yapmak
- Geometri ve veri kalitesi doğrulama sorguları yazmak
- Change Data Capture (CDC) ile artımlı yükleme uygulamak

---

## ETL Boru Hattı

Dört veri kaynağı entegre edilir:

1. Adres kayıt sistemi (CSV — UTM 32635)
2. Transit API (JSON — WGS84)
3. Su sayacı veritabanı (PostgreSQL — WGS84)
4. Atık sistemi (Excel — coğrafi kodlu)

---

### Veri çıkarma (Extraction)

<RunnableBlock
  sql={`-- Adres kayıt sistemi staging tablosu
CREATE TABLE IF NOT EXISTS addresses_staging (
  id BIGINT,
  street VARCHAR(255),
  number VARCHAR(50),
  district VARCHAR(100),
  postal_code VARCHAR(10),
  x_utm NUMERIC,
  y_utm NUMERIC,
  created_at TIMESTAMP
);

-- Örnek kayıtlar (UTM 32635 koordinatları)
INSERT INTO addresses_staging VALUES
  (1, 'İstiklal Cad', '100', 'Beyoğlu', '34433', 388000, 4543000, NOW()),
  (2, 'Bağdat Cad', '50', 'Kadıköy', '34740', 470000, 4527000, NOW())
ON CONFLICT DO NOTHING;

SELECT COUNT(*) AS aktarilan_adres FROM addresses_staging;`}
  label="Adres veri çıkarma"
  description="Staging tablosu kaynak sistemdeki ham veriyi aynen depolar; dönüşüm bu aşamada yapılmaz. x_utm ve y_utm sütunları EPSG:32635 (UTM Bölge 35N) koordinatlarını depolar; bu projeksiyon Türkiye'nin büyük kısmını kapsar. ON CONFLICT DO NOTHING tekrarlanan yüklemelerde veri çakışmasını önler."

### Koordinat dönüşümü (Transformation)

<RunnableBlock
  sql={`-- UTM 32635 → WGS84 4326 dönüşümü
WITH addresses_transformed AS (
  SELECT 
    id,
    street,
    district,
    ST_Transform(
      ST_SetSRID(ST_MakePoint(x_utm, y_utm), 32635),
      4326
    ) AS geom_wgs84
  FROM addresses_staging
)
SELECT 
  id,
  street,
  district,
  ROUND(ST_X(geom_wgs84)::numeric, 6) AS boylam,
  ROUND(ST_Y(geom_wgs84)::numeric, 6) AS enlem,
  'EPSG:4326' AS koordinat_sistemi
FROM addresses_transformed
LIMIT 5;`}
  label="Koordinat dönüşümü"
  description="ST_SetSRID(ST_MakePoint(x, y), 32635) ham UTM koordinatlarından kaynak SRID belirlenmiş bir POINT üretir. ST_Transform(..., 4326) bunu WGS84 coğrafi koordinatlarına dönüştürür. Bu dönüşüm kayıp verisi önlemek için doğru kaynak SRID'in bilinmesini gerektirir; yanlış SRID tüm noktaların coğrafi olarak hatalı konumlandırılmasına yol açar."
/>

### Veri kalitesi doğrulama

<RunnableBlock
  sql={`-- Kalite kontrol sorguları
WITH validation AS (
  SELECT 
    'NULL geometriler' AS kontrol_tipi,
    COUNT(*) FILTER (WHERE x_utm IS NULL OR y_utm IS NULL) AS basarisiz,
    COUNT(*) AS toplam
  FROM addresses_staging
  UNION ALL
  SELECT 
    'Kapsam dışı koordinatlar',
    COUNT(*) FILTER (WHERE x_utm < 200000 OR x_utm > 800000) AS basarisiz,
    COUNT(*) AS toplam
  FROM addresses_staging
  UNION ALL
  SELECT 
    'Tekrarlanan kayıtlar',
    COUNT(*) - COUNT(DISTINCT id) AS basarisiz,
    COUNT(*) AS toplam
  FROM addresses_staging
)
SELECT 
  kontrol_tipi,
  basarisiz,
  toplam,
  ROUND((basarisiz::numeric / NULLIF(toplam, 0) * 100), 2) AS hata_orani_pct,
  CASE 
    WHEN basarisiz = 0 THEN 'GECTI'
    ELSE 'BASARISIZ'
  END AS sonuc
FROM validation;`}
  label="Veri kalitesi doğrulama"
  description="Üç kalite kontrolü uygulanır: NULL koordinat varlığı, UTM Zone 35N sınırları dışında kalan değerler (bölge dışı koordinat), ve tekrarlanan kayıt kimliği. COUNT FILTER ile her kural ayrı ayrı sayılır. hata_orani_pct = basarisiz / toplam * 100 yüzde hata oranını gösterir. %99.5 kalite hedefi üretim ortamına geçiş kriteri olarak kullanılır."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Change Data Capture (artımlı yükleme) deseni
CREATE TABLE IF NOT EXISTS addresses_log (
  id BIGINT,
  operation VARCHAR(10),
  old_values JSONB,
  new_values JSONB,
  log_timestamp TIMESTAMP
);

-- Örnek CDC olayları
INSERT INTO addresses_log VALUES
  (1, 'INSERT', NULL, '{"street":"İstiklal"}', NOW()),
  (2, 'UPDATE', '{"district":"Fatih"}', '{"district":"Beyoğlu"}', NOW() + INTERVAL '1 minute'),
  (3, 'DELETE', '{"id":3}', NULL, NOW() + INTERVAL '2 minutes')
ON CONFLICT DO NOTHING;

-- Günlük delta özeti
SELECT 
  operation,
  COUNT(*) AS islem_sayisi,
  'Bugünkü değişiklikler' AS toplu_yukleme
FROM addresses_log
WHERE log_timestamp >= CURRENT_DATE
GROUP BY operation
ORDER BY operation;`}
  label="Artımlı yükleme (CDC)"
  description="Change Data Capture deseni yalnızca değişen kayıtları işleyerek tam yükleme maliyetini ortadan kaldırır. INSERT/UPDATE/DELETE işlemleri old_values ve new_values JSONB sütunlarıyla günlüklenir; bu yapı hangi alanın nasıl değiştiğini izlemeye olanak tanır. log_timestamp >= CURRENT_DATE filtresi günlük delta raporlamasında kullanılır. Üretimde bu yapı PostgreSQL logical replication veya trigger tabanlı yaklaşımlarla beslenir."
/>
