## Öğrenim Hedefleri

- `ST_ClusterDBSCAN` parametrelerini doğru yapılandırmak
- Gürültü noktaları (`cid = NULL`) ile küme noktaları arasındaki farkı yorumlamak
- Kara nokta analizi (hotspot) gerçekleştirmek

---

## DBSCAN Algoritması

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) iki parametre ile çalışır:
- **eps:** Komşuluk yarıçapı — bu mesafe içindeki noktalar komşu sayılır
- **min_points:** Bir küme oluşturmak için gereken minimum nokta sayısı

Yeterli yoğunluktaki noktalar küme oluşturur; geri kalanlar **gürültü** olarak sınıflandırılır (`cid = NULL`).

```sql
ST_ClusterDBSCAN(geom, eps := 500, minpoints := 5) OVER ()
-- eps: geometry biriminde → 4326 koordinatlarında derece!
-- Metre için: önce ST_Transform(geom, 3857) yap
```

---

## Geometri → Metre Dönüşümü

`ST_ClusterDBSCAN` geometry biriminde çalışır. EPSG:4326'da 1 birim ≈ 111 km olduğundan doğru metre hesabı için koordinat dönüşümü zorunludur:

```sql
-- Metre cinsinden eps: önce 3857'ye dönüştür
ST_ClusterDBSCAN(ST_Transform(geom, 3857), eps := 500, minpoints := 5) OVER ()
```

## Uygulama Örneği

### Kara noktalar

<RunnableBlock
  sql={`-- 500m içinde en az 5 kaza: kara nokta kümesi
SELECT
  cid,
  COUNT(*) AS kaza_sayisi,
  ST_AsText(ST_Centroid(ST_Collect(geom))) AS merkez_wkt,
  ST_Centroid(ST_Collect(geom)) AS geom
FROM (
  SELECT
    ST_ClusterDBSCAN(
      ST_Transform(geom, 3857),
      eps := 500,
      minpoints := 5
    ) OVER () AS cid,
    geom
  FROM kaza_noktalari
) sub
WHERE cid IS NOT NULL  -- gürültüyü dışla
GROUP BY cid
ORDER BY kaza_sayisi DESC;`}
  label="DBSCAN kara noktalar"
  description="ST_Transform(geom, 3857) koordinatları metre cinsine çevirir; eps := 500 500 metre yarıçaplı komşuluk tanımlar. minpoints := 5 en az 5 kazanın bulunduğu bölgeler küme olarak kabul edilir. WHERE cid IS NOT NULL gürültü noktaları filtreler. Sonuç her kara noktanın merkez koordinatını ve kaza sayısını listeler."
/>

### Küme vs gürültü dağılımı

<RunnableBlock
  sql={`SELECT

  COUNT(*) AS nokta_sayisi

  SELECT

### Parametre karşılaştırması

<RunnableBlock
  sql={`-- eps ve minpoints değiştirince küme sayısı nasıl değişiyor?
SELECT
  eps_m,
  minpts,
  COUNT(DISTINCT cid) FILTER (WHERE cid IS NOT NULL) AS kume_sayisi,
  COUNT(*) FILTER (WHERE cid IS NULL) AS gurultu_sayisi
FROM (
  SELECT 200 AS eps_m, 3 AS minpts,
    ST_ClusterDBSCAN(ST_Transform(geom,3857), 200, 3) OVER () AS cid
  FROM kaza_noktalari
  UNION ALL
  SELECT 500, 5,
    ST_ClusterDBSCAN(ST_Transform(geom,3857), 500, 5) OVER () AS cid
  FROM kaza_noktalari
  UNION ALL
  SELECT 1000, 10,
    ST_ClusterDBSCAN(ST_Transform(geom,3857), 1000, 10) OVER () AS cid
  FROM kaza_noktalari
) t
GROUP BY eps_m, minpts
ORDER BY eps_m;`}
  label="Parametre karşılaştırması"
  description="Üç farklı parametre kombinasyonu (200m/3pt, 500m/5pt, 1000m/10pt) UNION ALL ile tek sorguda karşılaştırılır. COUNT(DISTINCT cid) FILTER (WHERE cid IS NOT NULL) küme sayısını, COUNT(*) FILTER (WHERE cid IS NULL) gürültü sayısını hesaplar. eps büyüdükçe kümeler birleşir; minpoints yükseldikçe gürültü artar."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Sadece ağır kazalar için hotspot — daha sıkı parametre
SELECT
  cid,
  COUNT(*) AS agir_kaza,
  ST_Centroid(ST_Collect(geom)) AS geom
FROM (
  SELECT
    ST_ClusterDBSCAN(
      ST_Transform(geom, 3857), 1000, 3
    ) OVER () AS cid,
    geom
  FROM kaza_noktalari
  WHERE agir = TRUE
) sub
WHERE cid IS NOT NULL
GROUP BY cid
ORDER BY agir_kaza DESC
LIMIT 5;`}
  label="Ağır kaza hotspot top-5"
  description="WHERE agir = TRUE filtresi yalnızca ağır kazaları seçer. 1000 metre yarıçapında en az 3 ağır kaza barındıran bölgeler kara nokta olarak tespit edilir. Sonuç en çok ağır kaza içeren 5 hotspot noktasını harita koordinatlarıyla birlikte döndürür; bu bölgeler trafik müdahale öncelik sıralamasında kullanılabilir."
/>

## Yaygın Hatalar

1. **4326'da eps derece cinsinden:** `eps := 0.005` ekvatorda ≈ 500 m olur; ancak Türkiye enleminde farklıdır. Her zaman 3857 veya UTM projeksiyonuna dönüştürmek gerekir.
2. **cid NULL = gürültü:** KMeans'ta hiçbir nokta NULL değildir. DBSCAN'da gürültüler `cid IS NULL` döndürür; WHERE filtresi olmadan sonuçlar karışır.
3. **minpoints=1:** Her nokta kendi kümesi haline gelir ve analiz anlamsızlaşır. Minimum 3–5 değeri önerilir.
