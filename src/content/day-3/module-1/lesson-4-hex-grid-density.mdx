## Öğrenim Hedefleri

- Hexagonal grid ile yoğunluk haritası oluşturmak
- `ST_SquareGrid` ve `ST_HexagonGrid` fonksiyonlarını uygulamak
- Grid hücresi başına nokta yoğunluğunu hesaplamak

---

## Grid Tabanlı Yoğunluk

H3 (Uber'in hexagonal grid sistemi) PGlite ortamında mevcut değildir; ancak PostGIS'in kendi grid fonksiyonları aynı amaca hizmet eder:

| Fonksiyon | Açıklama |
|---|---|
| `ST_SquareGrid(size, bounds)` | Kare grid üretir |
| `ST_HexagonGrid(size, bounds)` | Altıgen grid üretir |
| `ST_PixelAsPolygons` | Raster tabanlı grid |

Bu fonksiyonlar belirli bir alan üzerine eşit büyüklükte hücreler üretir. Her hücreye düşen nokta sayısı yoğunluk değerini verir.

---

## Uygulama Örneği

### Kare grid yoğunluk haritası

<RunnableBlock
  sql={`-- İstanbul bbox üzerine 0.05 derece (~5 km) kare grid
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.5, 40.8, 29.5, 41.3, 4326) AS geom
),
grid AS (
  SELECT g.geom
  FROM bbox, ST_SquareGrid(0.05, bbox.geom) g
)
SELECT
  g.geom,
  COUNT(k.id) AS kaza_sayisi
FROM grid g
LEFT JOIN kaza_noktalari k
  ON ST_Within(k.geom, g.geom)
GROUP BY g.geom
HAVING COUNT(k.id) > 0
ORDER BY kaza_sayisi DESC
LIMIT 20;`}
  label="Kare grid yoğunluk haritası"
  description="ST_MakeEnvelope İstanbul çevresini sınırlayan dikdörtgen oluşturur. ST_SquareGrid(0.05, bbox) bu alan üzerine ~5 km kenar uzunluklu kare hücreler üretir. ST_Within her kazanın hangi hücrede olduğunu belirler. HAVING COUNT(k.id) > 0 boş hücreleri filtreler. Sonuç en kalabalık 20 hücreyi poligon geometrisiyle haritada gösterir."
/>

### Altıgen grid yoğunluk

<RunnableBlock
  sql={`-- Altıgen grid — H3'e benzer görünüm
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.5, 40.8, 29.5, 41.3, 4326) AS geom
),
hexgrid AS (
  SELECT h.geom
  FROM bbox, ST_HexagonGrid(0.05, bbox.geom) h
)
SELECT
  h.geom,
  COUNT(k.id) AS kaza_sayisi,
  CASE
    WHEN COUNT(k.id) >= 20 THEN 'yüksek'
    WHEN COUNT(k.id) >= 10 THEN 'orta'
    WHEN COUNT(k.id) >= 1  THEN 'düşük'
    ELSE 'yok'
  END AS yogunluk
FROM hexgrid h
LEFT JOIN kaza_noktalari k
  ON ST_Within(k.geom, h.geom)
GROUP BY h.geom
HAVING COUNT(k.id) > 0
ORDER BY kaza_sayisi DESC;`}
  label="Altıgen grid yoğunluk"
  description="ST_HexagonGrid kare yerine altıgen hücreler üretir; altıgenler komşu hücrelere eşit uzaklık sağlar ve görsel olarak H3 sistemine benzer. CASE ifadesi kaza sayısına göre her hücreyi 'yüksek/orta/düşük' olarak kategorize eder. geom sütunu haritada renkli poligonlar olarak görünür."
/>

### km² başına yoğunluk

<RunnableBlock
  sql={`-- Alanı bilinen grid hücreleri için km² başına yoğunluk
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.5, 40.8, 29.5, 41.3, 4326) AS geom
),
grid AS (
  SELECT g.geom,
    ROUND(ST_Area(g.geom::geography)::numeric / 1000000, 2) AS alan_km2
  FROM bbox, ST_SquareGrid(0.05, bbox.geom) g
)
SELECT
  g.geom,
  g.alan_km2,
  COUNT(k.id) AS kaza_sayisi,
  ROUND(COUNT(k.id) / NULLIF(g.alan_km2, 0), 2) AS kaza_per_km2
FROM grid g
LEFT JOIN kaza_noktalari k ON ST_Within(k.geom, g.geom)
GROUP BY g.geom, g.alan_km2
HAVING COUNT(k.id) > 0
ORDER BY kaza_per_km2 DESC
LIMIT 10;`}
  label="km² başına kaza yoğunluğu"
  description="ST_Area(geom::geography) her hücrenin gerçek alanını metre kare cinsinden hesaplar; bölü 1.000.000 ile km²'ye çevrilir. NULLIF(g.alan_km2, 0) sıfıra bölme hatasını önler. kaza_per_km2 farklı boyuttaki hücreleri karşılaştırmayı sağlayan normalize edilmiş yoğunluk değeridir. En yüksek yoğunluklu 10 hücre öncelikli müdahale alanlarıdır."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Daha ince grid: 0.02 derece (~2 km) — daha hassas
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.8, 40.9, 29.2, 41.2, 4326) AS geom
)
SELECT COUNT(*) AS hucre_sayisi
FROM bbox, ST_SquareGrid(0.02, bbox.geom) g
JOIN kaza_noktalari k ON ST_Within(k.geom, g.geom)
GROUP BY g.geom
HAVING COUNT(k.id) > 3;`}
  label="İnce grid hotspot hücreler"
  description="0.02 derece (~2 km) kenar uzunluğu daha ince çözünürlüklü analiz sağlar. JOIN kaza_noktalari ON ST_Within boş hücreleri zaten dışarıda bırakır; HAVING > 3 ek filtre ile yalnızca 3'ten fazla kaza içeren yoğun hücreler sayılır. Grid boyutunu küçültmek hassasiyeti artırır ancak JOIN maliyetini de yükseltir."
/>

## Yaygın Hatalar

1. **`ST_SquareGrid` / `ST_HexagonGrid` PostGIS 3.1+ gerektirir:** Eski sürümlerde bu fonksiyonlar bulunmaz. Desteklenmiyorsa `generate_series` ile elle grid oluşturulabilir.
2. **Grid hücresi çok küçükse:** Binlerce hücre ve binlerce JOIN işlemi performansı düşürür. Grid boyutu veri yoğunluğuyla orantılı seçilmelidir.
3. **Sınır kenarında kesik hücreler:** `ST_Intersection(g.geom, bbox)` ile sınır kenarındaki kısmi hücreler kırpılır ve kenar etkisi azaltılır.
