## 🎯 Ne öğreneceksin?

- Hexagonal grid ile yoğunluk haritası oluşturabileceksin
- `ST_SquareGrid` ve `ST_HexagonGrid` fonksiyonlarını kullanabileceksin
- Grid hücresi başına nokta sayısını hesaplayabileceksin

---

## 📖 Kavram: Grid Tabanlı Yoğunluk

H3 (Uber'in hexagonal grid sistemi) PGlite'ta mevcut değil — ama PostGIS'in kendi grid fonksiyonları var:

| Fonksiyon | Açıklama |
|---|---|
| `ST_SquareGrid(size, bounds)` | Kare grid üret |
| `ST_HexagonGrid(size, bounds)` | Altıgen grid üret |
| `ST_PixelAsPolygons` | Raster tabanlı grid |

Bu fonksiyonlar bir alan üzerine eşit büyüklükte hücreler üretir. Her hücreye düşen nokta sayısı → yoğunluk.

---

## ▶ Canlı Örnek — Kare Grid Yoğunluk

<RunnableBlock
  sql={`-- İstanbul bbox üzerine 0.05 derece (~5 km) kare grid
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.5, 40.8, 29.5, 41.3, 4326) AS geom
),
grid AS (
  SELECT g.geom
  FROM bbox, ST_SquareGrid(0.05, bbox.geom) g
)
SELECT
  g.geom,
  COUNT(k.id) AS kaza_sayisi
FROM grid g
LEFT JOIN kaza_noktalari k
  ON ST_Within(k.geom, g.geom)
GROUP BY g.geom
HAVING COUNT(k.id) > 0
ORDER BY kaza_sayisi DESC
LIMIT 20;`}
  label="Kare grid yoğunluk haritası"
/>

## ▶ Altıgen Grid

<RunnableBlock
  sql={`-- Altıgen grid — H3'e benzer görünüm
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.5, 40.8, 29.5, 41.3, 4326) AS geom
),
hexgrid AS (
  SELECT h.geom
  FROM bbox, ST_HexagonGrid(0.05, bbox.geom) h
)
SELECT
  h.geom,
  COUNT(k.id) AS kaza_sayisi,
  CASE
    WHEN COUNT(k.id) >= 20 THEN 'yüksek'
    WHEN COUNT(k.id) >= 10 THEN 'orta'
    WHEN COUNT(k.id) >= 1  THEN 'düşük'
    ELSE 'yok'
  END AS yogunluk
FROM hexgrid h
LEFT JOIN kaza_noktalari k
  ON ST_Within(k.geom, h.geom)
GROUP BY h.geom
HAVING COUNT(k.id) > 0
ORDER BY kaza_sayisi DESC;`}
  label="Altıgen grid yoğunluk"
/>

## ▶ Yoğunluk Sütununu Hesapla

<RunnableBlock
  sql={`-- Alanı bilinen grid hücreleri için km² başına yoğunluk
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.5, 40.8, 29.5, 41.3, 4326) AS geom
),
grid AS (
  SELECT g.geom,
    ROUND(ST_Area(g.geom::geography)::numeric / 1000000, 2) AS alan_km2
  FROM bbox, ST_SquareGrid(0.05, bbox.geom) g
)
SELECT
  g.geom,
  g.alan_km2,
  COUNT(k.id) AS kaza_sayisi,
  ROUND(COUNT(k.id) / NULLIF(g.alan_km2, 0), 2) AS kaza_per_km2
FROM grid g
LEFT JOIN kaza_noktalari k ON ST_Within(k.geom, g.geom)
GROUP BY g.geom, g.alan_km2
HAVING COUNT(k.id) > 0
ORDER BY kaza_per_km2 DESC
LIMIT 10;`}
  label="km² başına kaza yoğunluğu"
/>

## 🧪 Dene — Grid Boyutunu Değiştir

<RunnableBlock
  sql={`-- Daha ince grid: 0.02 derece (~2 km) — daha hassas
WITH bbox AS (
  SELECT ST_MakeEnvelope(28.8, 40.9, 29.2, 41.2, 4326) AS geom
)
SELECT COUNT(*) AS hucre_sayisi
FROM bbox, ST_SquareGrid(0.02, bbox.geom) g
JOIN kaza_noktalari k ON ST_Within(k.geom, g.geom)
GROUP BY g.geom
HAVING COUNT(k.id) > 3;`}
  label="İnce grid hotspot hücreler"
/>

## ⚠ Tuzaklar

1. **`ST_SquareGrid` / `ST_HexagonGrid` PostGIS 3.1+ gerektirir:** PGlite PostGIS sürümüne bağlı — çalışmazsa `generate_series` ile elle grid oluştur.
2. **Grid hücresi çok küçükse:** Binlerce hücre, binlerce JOIN — performans düşer. Grid boyutunu veri yoğunluğuna göre ayarla.
3. **Sınır kenarında kesik hücreler:** `ST_Intersection(g.geom, bbox)` ile kesilen hücreleri kıs — kenar etkileri azalır.
