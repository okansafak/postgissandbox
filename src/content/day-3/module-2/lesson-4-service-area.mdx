## 🎯 Ne öğreneceksin?

- Servis alanı (izokon) kavramını açıklayabileceksin
- Ağ mesafesiyle ulaşılabilir düğümleri bulabileceksin
- `ST_Buffer` ile yaklaşık izokon çizebileceksin

---

## 📖 Kavram: Servis Alanı (İzokon)

"Bir itfaiye istasyonundan 2 km içinde hangi alanlar var?"

**İki yaklaşım:**
1. **Ağ mesafesi:** Yol üzerinden 2 km — gerçekçi ama hesap yoğun
2. **Kuş uçuşu buffer:** Merkez noktasından 2 km radius — hızlı yaklaşım

pgRouting'de `pgr_drivingDistance` ağ mesafesiyle servis alanı üretir. PGlite'ta bunu recursive CTE simülasyonuyla göstereceğiz.

---

## ▶ Canlı Örnek — Ağ Mesafesiyle Ulaşılabilir Düğümler

<RunnableBlock
  sql={`-- Taksim'den 5 km mesafede ulaşılabilir düğümler
WITH RECURSIVE servis_alani AS (
  -- Başlangıç: Taksim (düğüm 1)
  SELECT 1 AS dugum, 0.0 AS mesafe_m
  UNION ALL
  SELECT
    CASE WHEN y.source = s.dugum THEN y.target ELSE y.source END AS sonraki,
    s.mesafe_m + y.uzunluk_m
  FROM servis_alani s
  JOIN yol_agi y ON (y.source = s.dugum OR (y.target = s.dugum AND NOT y.tek_yon))
  WHERE s.mesafe_m + y.uzunluk_m <= 5000  -- 5 km sınırı
),
erisim AS (
  SELECT dugum, MIN(mesafe_m) AS en_kisa
  FROM servis_alani
  GROUP BY dugum
)
SELECT
  e.dugum,
  d.ad AS dugum_adi,
  ROUND(e.en_kisa::numeric) AS mesafe_m,
  d.geom
FROM erisim e
JOIN yol_dugumler d ON d.id = e.dugum
ORDER BY e.en_kisa;`}
  label="5 km servis alanı düğümleri"
/>

## ▶ Buffer ile Görsel Servis Alanı

<RunnableBlock
  sql={`-- Ulaşılabilir düğümlerin convex hull = yaklaşık servis alanı
WITH RECURSIVE servis_alani AS (
  SELECT 1 AS dugum, 0.0 AS mesafe_m
  UNION ALL
  SELECT
    CASE WHEN y.source = s.dugum THEN y.target ELSE y.source END,
    s.mesafe_m + y.uzunluk_m
  FROM servis_alani s
  JOIN yol_agi y ON (y.source = s.dugum OR (y.target = s.dugum AND NOT y.tek_yon))
  WHERE s.mesafe_m + y.uzunluk_m <= 5000
),
erisim AS (
  SELECT dugum, MIN(mesafe_m) AS en_kisa FROM servis_alani GROUP BY dugum
),
erisim_noktalar AS (
  SELECT d.geom FROM erisim e JOIN yol_dugumler d ON d.id = e.dugum
)
SELECT
  -- Convex hull: erişilen noktaların dış çerçevesi
  ST_ConvexHull(ST_Collect(geom)) AS servis_alani_geom,
  COUNT(*) AS erisilen_dugum
FROM erisim_noktalar;`}
  label="Servis alanı convex hull"
/>

## ▶ ST_Buffer ile İzokon Karşılaştırması

<RunnableBlock
  sql={`-- Kuş uçuşu vs ağ mesafesi karşılaştırması
-- Taksim merkez: kuş uçuşu 3 km buffer
SELECT
  ST_Buffer(
    ST_SetSRID(ST_MakePoint(28.979, 41.037), 4326)::geography,
    3000
  )::geometry AS buffer_geom,
  'kuş uçuşu 3km' AS yontem;`}
  label="Kuş uçuşu buffer izokon"
/>

> Ağ mesafesi genellikle buffer'dan daha küçük alan kapsar — özellikle nehir, demiryolu ve çıkmaz sokaklar nedeniyle.

## 🧪 Dene — Farklı Başlangıç

<RunnableBlock
  sql={`-- Levent'ten (düğüm 4) erişilebilir yerler
WITH RECURSIVE servis AS (
  SELECT 4 AS dugum, 0.0 AS mesafe_m
  UNION ALL
  SELECT
    CASE WHEN y.source = s.dugum THEN y.target ELSE y.source END,
    s.mesafe_m + y.uzunluk_m
  FROM servis s
  JOIN yol_agi y ON (y.source = s.dugum OR (y.target = s.dugum AND NOT y.tek_yon))
  WHERE s.mesafe_m + y.uzunluk_m <= 4000
)
SELECT d.ad, ROUND(MIN(s.mesafe_m)::numeric) AS en_kisa_m
FROM servis s JOIN yol_dugumler d ON d.id = s.dugum
GROUP BY d.ad ORDER BY en_kisa_m;`}
  label="Levent 4 km servis alanı"
/>

## ⚠ Tuzaklar

1. **Büyük ağda recursive CTE sonsuz tekrar:** `mesafe_m <= limit` koşulu olmadan sonsuz döngü. Limit her zaman koy.
2. **Aynı düğüme farklı yollar:** `MIN(mesafe_m)` ile en kısa mesafeyi al — aksi halde aynı düğüm birden fazla kez döner.
3. **İzokon ≠ buffer:** Buffer dairesel — gerçek servis alanı yol yapısına bağlı. pgr_drivingDistance + alpha-shape daha doğru sonuç verir.
