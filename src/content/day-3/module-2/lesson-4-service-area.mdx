## Öğrenim Hedefleri

- Servis alanı (izokon) kavramını açıklamak
- Ağ mesafesiyle ulaşılabilir düğümleri bulmak
- `ST_Buffer` ile yaklaşık izokon oluşturmak

---

## Servis Alanı (İzokon)

"Bir itfaiye istasyonundan 2 km içinde hangi alanlar var?" sorusu servis alanı analizinin temel sorusudur.

**İki yaklaşım:**
1. **Ağ mesafesi:** Yol üzerinden 2 km — gerçekçi ancak hesap yoğun
2. **Kuş uçuşu buffer:** Merkez noktasından 2 km yarıçaplı alan — hızlı yaklaşım

pgRouting'de `pgr_drivingDistance` ağ mesafesiyle servis alanı üretir. PGlite'ta bu işlev recursive CTE simülasyonuyla gösterilir.

---

## Uygulama Örneği

### 5 km servis alanı düğümleri

<RunnableBlock
  sql={`-- Taksim'den 5 km mesafede ulaşılabilir düğümler
WITH RECURSIVE servis_alani AS (
  -- Başlangıç: Taksim (düğüm 1)
  SELECT 1 AS dugum, 0.0 AS mesafe_m
  UNION ALL
  SELECT
    CASE WHEN y.source = s.dugum THEN y.target ELSE y.source END AS sonraki,
    s.mesafe_m + y.uzunluk_m
  FROM servis_alani s
  JOIN yol_agi y ON (y.source = s.dugum OR (y.target = s.dugum AND NOT y.tek_yon))
  WHERE s.mesafe_m + y.uzunluk_m <= 5000  -- 5 km sınırı
),
erisim AS (
  SELECT dugum, MIN(mesafe_m) AS en_kisa
  FROM servis_alani
  GROUP BY dugum
)
SELECT
  e.dugum,
  d.ad AS dugum_adi,
  ROUND(e.en_kisa::numeric) AS mesafe_m,
  d.geom
FROM erisim e
JOIN yol_dugumler d ON d.id = e.dugum
ORDER BY e.en_kisa;`}
  label="5 km servis alanı düğümleri"
  description="Taksim'den (dugum=1) başlayan recursive CTE 5000 metre sınırı içindeki tüm ulaşılabilir düğümleri genişletir. MIN(mesafe_m) GROUP BY ile her düğüme en kısa mesafe hesaplanır; aynı düğüme farklı yollarla ulaşılmış olsa bile en kısa mesafe korunur. geom sütunu düğüm noktalarını haritada gösterir."
/>

### Servis alanı convex hull

<RunnableBlock
  sql={`-- Ulaşılabilir düğümlerin convex hull = yaklaşık servis alanı
WITH RECURSIVE servis_alani AS (
  SELECT 1 AS dugum, 0.0 AS mesafe_m
  UNION ALL
  SELECT
    CASE WHEN y.source = s.dugum THEN y.target ELSE y.source END,
    s.mesafe_m + y.uzunluk_m
  FROM servis_alani s
  JOIN yol_agi y ON (y.source = s.dugum OR (y.target = s.dugum AND NOT y.tek_yon))
  WHERE s.mesafe_m + y.uzunluk_m <= 5000
),
erisim AS (
  SELECT dugum, MIN(mesafe_m) AS en_kisa FROM servis_alani GROUP BY dugum
),
erisim_noktalar AS (
  SELECT d.geom FROM erisim e JOIN yol_dugumler d ON d.id = e.dugum
)
SELECT
  ST_ConvexHull(ST_Collect(geom)) AS servis_alani_geom,
  COUNT(*) AS erisilen_dugum
FROM erisim_noktalar;`}
  label="Servis alanı convex hull"
  description="Ulaşılabilir tüm düğüm noktaları ST_Collect ile bir koleksiyona toplanır. ST_ConvexHull bu noktaları çevreleyen en küçük dışbükey poligonu oluşturur. Bu poligon yaklaşık servis alanını temsil eder. Gerçek servis alanı yol geometrisine göre daha düzensiz bir şekle sahiptir; alpha-shape algoritması daha doğru sınır verir."
/>

### Kuş uçuşu buffer izokon

<RunnableBlock
  sql={`-- Kuş uçuşu vs ağ mesafesi karşılaştırması
-- Taksim merkez: kuş uçuşu 3 km buffer
SELECT
  ST_Buffer(
    ST_SetSRID(ST_MakePoint(28.979, 41.037), 4326)::geography,
    3000
  )::geometry AS buffer_geom,
  'kuş uçuşu 3km' AS yontem;`}
  label="Kuş uçuşu buffer izokon"
  description="ST_Buffer geography üzerinde çalışarak gerçek dairesel alan oluşturur; 3000 metre yarıçaplı alan Taksim merkezli olarak hesaplanır. ::geometry cast'i sonucu harita görünümüne uygun formata çevirir. Ağ mesafesi genellikle bu buffer'dan daha küçük alan kapsar; nehir, demiryolu ve çıkmaz sokaklar ağ erişimini kısıtlar."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Levent'ten (düğüm 4) erişilebilir yerler
WITH RECURSIVE servis AS (
  SELECT 4 AS dugum, 0.0 AS mesafe_m
  UNION ALL
  SELECT
    CASE WHEN y.source = s.dugum THEN y.target ELSE y.source END,
    s.mesafe_m + y.uzunluk_m
  FROM servis s
  JOIN yol_agi y ON (y.source = s.dugum OR (y.target = s.dugum AND NOT y.tek_yon))
  WHERE s.mesafe_m + y.uzunluk_m <= 4000
)
SELECT d.ad, ROUND(MIN(s.mesafe_m)::numeric) AS en_kisa_m
FROM servis s JOIN yol_dugumler d ON d.id = s.dugum
GROUP BY d.ad ORDER BY en_kisa_m;`}
  label="Levent 4 km servis alanı"
  description="Levent'ten (dugum=4) 4000 metre sınırıyla aynı analiz tekrarlanır. TEM Bağlantı kenarı tek yönlü (Levent → Maslak) olduğundan Maslak erişilebilir; ancak Maslak'tan Levent'e bu yoldan geçilemez. MIN ile aynı düğüme en kısa mesafe seçilir."
/>

## Yaygın Hatalar

1. **Büyük ağda recursive CTE sonsuz tekrar:** `mesafe_m <= limit` koşulu olmadan sonsuz döngüye girilir; limit her zaman konmalıdır.
2. **Aynı düğüme farklı yollar:** `MIN(mesafe_m)` ile en kısa mesafe seçilmelidir; aksi halde aynı düğüm birden fazla kez döner.
3. **İzokon ≠ buffer:** Buffer daireseldir; gerçek servis alanı yol yapısına bağlı biçimsiz bir alan kapsar. `pgr_drivingDistance` + alpha-shape daha doğru sonuç verir.
