## Öğrenim Hedefleri

- Dijkstra algoritmasının işleyişini açıklamak
- SQL recursive CTE ile en kısa yolu hesaplamak
- `yol_agi` tablosu üzerinde A → B güzergahı bulmak

---

## Dijkstra Algoritması

1. Başlangıç düğümüne mesafe = 0, diğerlerine ∞ atanır
2. En düşük maliyetli düğüm seçilir
3. Komşularına güncellenmiş mesafe yazılır (geçiş daha ucuzsa)
4. Hedef düğüme ulaşılana kadar tekrarlanır

**SQL ile:** Recursive CTE (WITH RECURSIVE) Dijkstra'yı yaklaşık olarak simüle eder; küçük ağlarda yeterlidir.

## Uygulama Örneği

### Taksim'den Maslak'a

<RunnableBlock
  sql={`-- Yol ağı tablosunun mevcut olduğunu varsayıyoruz (lesson-1'den)
-- En kısa yol: Düğüm 1 (Taksim) → Düğüm 5 (Maslak)
WITH RECURSIVE dijkstra AS (
  -- Başlangıç: Taksim (düğüm 1)
  SELECT
    1 AS dugum,
    0.0 AS toplam_maliyet,
    ARRAY[1] AS yol,
    FALSE AS ziyaret_edildi
  UNION ALL
  SELECT
    CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END,
    d.toplam_maliyet + y.uzunluk_m,
    d.yol || ARRAY[CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END],
    FALSE
  FROM dijkstra d
  JOIN yol_agi y ON (y.source = d.dugum OR (y.target = d.dugum AND NOT y.tek_yon))
  WHERE NOT (CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END = ANY(d.yol))
    AND array_length(d.yol, 1) < 6
)
SELECT
  yol,
  ROUND(toplam_maliyet::numeric / 1000, 2) AS toplam_km
FROM dijkstra
WHERE dugum = 5  -- Maslak
ORDER BY toplam_maliyet
LIMIT 3;`}
  label="Dijkstra: Taksim → Maslak"
  description="Recursive CTE Taksim'den (dugum=1) başlar ve her adımda komşu düğümlere genişler. yol dizisi ziyaret edilen düğümleri tutar; NOT ... = ANY(d.yol) döngüleri önler. array_length < 6 limiti sonsuz tekrarı durdurur. WHERE dugum = 5 yalnızca Maslak'a ulaşan yolları filtreler; ORDER BY toplam_maliyet en kısayı öne alır."
/>

### En kısa yol geometrisi

<RunnableBlock
  sql={`-- En kısa yolu haritada göster
WITH RECURSIVE dijkstra AS (
  SELECT 1 AS dugum, 0.0 AS maliyet, ARRAY[1] AS yol
  UNION ALL
  SELECT
    CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END,
    d.maliyet + y.uzunluk_m,
    d.yol || ARRAY[CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END]
  FROM dijkstra d
  JOIN yol_agi y ON (y.source = d.dugum OR (y.target = d.dugum AND NOT y.tek_yon))
  WHERE NOT (CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END = ANY(d.yol))
    AND array_length(d.yol, 1) < 6
),
en_kisa AS (
  SELECT yol FROM dijkstra WHERE dugum = 5 ORDER BY maliyet LIMIT 1
),
yol_segmentler AS (
  SELECT
    n1, n2,
    y.geom
  FROM (
    SELECT
      yol[i] AS n1,
      yol[i+1] AS n2
    FROM en_kisa, generate_series(1, array_length(yol, 1)-1) i
  ) pairs
  JOIN yol_agi y ON
    (y.source = n1 AND y.target = n2) OR
    (y.source = n2 AND y.target = n1)
)
SELECT ST_Collect(geom) AS rota_geom FROM yol_segmentler;`}
  label="En kısa yol geometrisi"
  description="En kısa güzergah düğüm ID dizisinden geometrik segmentlere dönüştürülür. generate_series ardışık düğüm çiftlerini (n1, n2) oluşturur. Her çifti karşılayan yol_agi satırı JOIN ile eşleştirilir. ST_Collect tüm segmentleri tek bir GeometryCollection'a toplar ve haritada çizgi olarak görünür."
/>

## Pratik Uygulama

<RunnableBlock
  sql={`-- Beşiktaş'tan Maslak'a (düğüm 2 → 5)
WITH RECURSIVE dijkstra AS (
  SELECT 2 AS dugum, 0.0 AS maliyet, ARRAY[2] AS yol
  UNION ALL
  SELECT
    CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END,
    d.maliyet + y.uzunluk_m,
    d.yol || ARRAY[CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END]
  FROM dijkstra d
  JOIN yol_agi y ON (y.source = d.dugum OR (y.target = d.dugum AND NOT y.tek_yon))
  WHERE NOT (CASE WHEN y.source = d.dugum THEN y.target ELSE y.source END = ANY(d.yol))
    AND array_length(d.yol, 1) < 6
)
SELECT
  yol,
  ROUND(maliyet::numeric / 1000, 2) AS km
FROM dijkstra
WHERE dugum = 5
ORDER BY maliyet
LIMIT 3;`}
  label="Beşiktaş → Maslak güzergahları"
  description="Başlangıç düğümü 2 (Beşiktaş) olarak değiştirilmiş aynı algoritma çalışır. İlk üç alternatif güzergah km cinsinden listelenir. Beşiktaş → Barbaros Bul → Levent → Maslak ve Beşiktaş → Taksim → Şişli → Levent → Maslak gibi alternatif rotaların toplam maliyeti karşılaştırılabilir."

## Yaygın Hatalar

1. **Recursive CTE Dijkstra = simülasyon:** Büyük ağlarda (10K+ kenar) bu yaklaşım yavaş ve tam doğru değildir. Gerçek projelerde pgRouting veya uygulama katmanı (OSRM, Valhalla) kullanılmalıdır.
2. **Döngü kontrolü:** `NOT ... = ANY(d.yol)` döngüleri önler; ancak `array_length < 6` limiti konmazsa sonsuz tekrara girilir.
3. **Tek yönlü yol:** `tek_yon = TRUE` olan kenarda geri yön geçişi engellenir. Tam yön kontrolü için `reverse_cost = -1` şeması incelenmelidir.
