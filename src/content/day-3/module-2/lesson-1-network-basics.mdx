## 📖 Kavram: Graf Teorisi ve Yol Ağı

Ağ analizi: Düğümler (kavşaklar) + Kenarlar (yollar) + Ağırlıklar (mesafe/süre) ile optimum güzergah bulma.

| Kavram | Yol ağında karşılığı |
|---|---|
| **Düğüm (node/vertex)** | Kavşak, dönüş noktası |
| **Kenar (edge)** | Yol segmenti |
| **Ağırlık (weight/cost)** | Mesafe, süre, ücret |
| **Yönlü graf** | Tek yönlü sokak |
| **Yönsüz graf** | İki yönlü yol |

**pgRouting:** PostgreSQL + PostGIS üzerine kurulu ağ analizi extension'ı. PGlite'ta resmi destek **yok** — bu modülde kavramsal anlatım + örnek veri üzerinde simülasyon yapacağız.

---

## 📖 Yol Ağı Tablo Şeması

pgRouting standart şeması:
```sql
CREATE TABLE yollar (
  id       BIGSERIAL PRIMARY KEY,
  source   BIGINT,  -- başlangıç düğüm ID
  target   BIGINT,  -- bitiş düğüm ID
  cost     FLOAT,   -- ileri yön maliyeti (mesafe/süre)
  reverse_cost FLOAT, -- geri yön (-1 = tek yönlü)
  geom     GEOMETRY(LINESTRING, 4326)
);
```

CREATE TABLE IF NOT EXISTS yol_dugumler (

  (1, 'Taksim',   ST_SetSRID(ST_MakePoint(28.979, 41.037),4326)),
  (2, 'Beşiktaş', ST_SetSRID(ST_MakePoint(29.005, 41.043),4326)),

  (5, 'Maslak',   ST_SetSRID(ST_MakePoint(29.020, 41.113),4326))
ON CONFLICT DO NOTHING;

-- Ağ istatistikleri
SELECT
  COUNT(DISTINCT source) + COUNT(DISTINCT target) AS benzersiz_dugum,
  COUNT(*) AS kenar_sayisi,
  ROUND(SUM(uzunluk_m)/1000, 1) AS toplam_km
FROM yol_agi;`}

## 🧪 Dene — Bağlantılı Komşular

<RunnableBlock
  sql={`-- Her düğümün komşuları
SELECT
  d.ad AS dugum,
  array_agg(DISTINCT k.ad ORDER BY k.ad) AS komsular
FROM yol_dugumler d
JOIN yol_agi y ON y.source = d.id OR y.target = d.id
JOIN yol_dugumler k ON (
  CASE WHEN y.source = d.id THEN y.target ELSE y.source END
) = k.id
GROUP BY d.id, d.ad
ORDER BY d.ad;`}

## ⚠ Tuzaklar

1. **pgRouting PGlite'ta yok:** `pgr_dijkstra` gibi fonksiyonlar çalışmaz. Sonraki derslerde recursive CTE ile Dijkstra simülasyonu yapacağız.
2. **source/target INT olmalı:** pgRouting standart şeması integer node ID bekler.
3. **Tek yönlü kenarlar:** `reverse_cost = -1` olunca pgRouting bu yönde geçiş yapmaz. Simülasyonda `tek_yon = TRUE` ile kontrol edeceğiz.
