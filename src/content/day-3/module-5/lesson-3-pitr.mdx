## 📖 Kavram: PITR

"Saat 14:30'da yanlışlıkla `DELETE FROM kadastro` çalıştırdım. 14:29'a dönebilir miyim?"

**Evet — eğer WAL arşivleme aktifse.**

PITR = Base Backup + WAL Segment Oynatma:
1. Son base backup'ı geri yükle
2. Hatalı işlemden önceki zamana kadar WAL segment'lerini oynat
3. `recovery_target_time` ile tam zaman belirle

---

## 📖 PITR Kurulum Gereksinimleri

```ini
# postgresql.conf
wal_level = replica
archive_mode = on
archive_command = 'cp %p /wal-arsiv/%f'
```

```bash
# Gece base backup
pg_basebackup -D /yedekler/$(date +%Y%m%d) -Ft -z -P

# Kurtarma: recovery.conf (PG 12+ postgresql.conf'ta)
restore_command = 'cp /wal-arsiv/%f %p'
recovery_target_time = '2024-01-15 14:29:00'
recovery_target_action = promote
```

## ⚠ Tuzaklar

1. **WAL arşivi doluysa:** Arşiv dolduğunda `archive_command` başarısız olur → PostgreSQL durabiliyor. Disk alanını izle.
2. **PITR + PostGIS:** Geri yükleme sonrası `SELECT postgis_full_version()` ile extension sağlığını kontrol et.
3. **`recovery_target_inclusive`:** Varsayılan `true` — hedef zamanı dahil. Hedef zamandaki kötü işlemi de oynatır. `false` yap veya bir saniye erken hedefle.
