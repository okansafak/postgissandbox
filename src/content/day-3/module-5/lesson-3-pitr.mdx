## Öğrenim Hedefleri

- PITR (Point-in-Time Recovery) kavramını açıklamak
- WAL arşivleme + base backup kombinasyonunu özetlemek
- Geri dönme senaryosunu adım adım anlatmak

---

## PITR

"Saat 14:30'da yanlışlıkla `DELETE FROM kadastro` çalıştırıldı. 14:29'a dönmek mümkün mü?"

**Evet — WAL arşivleme aktifse.**

PITR = Base Backup + WAL Segment Oynatma:
1. Son base backup geri yüklenir
2. Hatalı işlemden önceki zamana kadar WAL segmentleri oynatılır
3. `recovery_target_time` ile tam zaman belirlenir

---

## PITR Kurulum Gereksinimleri

```ini
# postgresql.conf
wal_level = replica
archive_mode = on
archive_command = 'cp %p /wal-arsiv/%f'
```

```bash
# Gece base backup
pg_basebackup -D /yedekler/$(date +%Y%m%d) -Ft -z -P

# Kurtarma: recovery.conf (PG 12+ postgresql.conf'ta)
restore_command = 'cp /wal-arsiv/%f %p'
recovery_target_time = '2024-01-15 14:29:00'
recovery_target_action = promote
```

## Yaygın Hatalar

1. **WAL arşivi doluysa:** Arşiv dolu olduğunda `archive_command` başarısız olur; PostgreSQL durabilir. Disk alanı sürekli izlenmelidir.
2. **PITR + PostGIS:** Geri yükleme sonrasında `SELECT postgis_full_version()` ile extension sağlığı doğrulanmalıdır.
3. **`recovery_target_inclusive`:** Varsayılan `true` — hedef zamanı dahil eder ve hedef zamandaki kötü işlemi de oynatır. Hatalı işlemi dışlamak için `false` ayarlanmalı veya bir saniye erken hedeflenmelidir.
