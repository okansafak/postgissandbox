## 📖 Kavram: Replikasyon Tipleri

| Özellik | Streaming | Logical |
|---|---|---|
| Kapsam | Tüm cluster (byte-by-byte) | Tablo seçimi mümkün |
| PostgreSQL versiyon farkı | Aynı major versiyon | Farklı versiyon OK |
| PostGIS desteği | Tam | Kısıtlı (`geometry` için logical decode eklentisi gerekebilir) |
| Read replica | Evet | Evet |
| Çift yönlü | Hayır | Evet (bi-directional) |
| Kullanım | HA, failover | Seçici sync, upgrade, cross-cluster |

---

## 📖 PostGIS ve Replikasyon

**Streaming Replication** PostGIS ile sorunsuz çalışır — fiziksel kopyalama yapar.

**Logical Replication** dikkat gerektiren durumlar:
- `geometry` tipi binary format — `pgoutput` plugin ile desteklenir (PG 15+)
- Extension yayınlamada `wal_level = logical` gerekir
- Subscriber tarafında extension kurulu olmalı

## 📖 Replikasyon Mimarisi

```
Üretim (Primary):
  PostgreSQL + PostGIS
  wal_level = replica (streaming) veya logical
       |
       | WAL stream
       ↓
Yedek (Standby):
  PostgreSQL + PostGIS (aynı versiyon)
  hot_standby = on → Read queries OK

Read Replica kullanım:
  Uygulama → Primary (yazma)
  Raporlama → Standby (okuma)
```

## ⚠ Tuzaklar

1. **Standby read-only:** Streaming standby yazma kabul etmez. Bağlantı pool'unu primary/standby olarak ikiye ayır.
2. **PostGIS extension standby'da:** Standby fiziksel kopya — extension otomatik mevcut. Logical replication'da manuel kurulum gerekir.
3. **Replikasyon lag izleme:** `pg_stat_replication.replay_lag` kolonunu izle. Uzun lag → standby yavaş → failover gecikmesi.
