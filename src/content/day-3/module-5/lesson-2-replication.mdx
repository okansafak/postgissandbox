## Öğrenim Hedefleri

- Streaming ve Logical Replication farkını açıklamak
- PostGIS için hangi replikasyon tipinin uygun olduğunu belirlemek
- Replikasyon kurulum adımlarını özetlemek

---

## Replikasyon Tipleri

| Özellik | Streaming | Logical |
|---|---|---|
| Kapsam | Tüm cluster (byte-by-byte) | Tablo seçimi mümkün |
| PostgreSQL versiyon farkı | Aynı major versiyon | Farklı versiyon desteklenir |
| PostGIS desteği | Tam | Kısıtlı (`geometry` için ek eklenti gerekebilir) |
| Read replica | Evet | Evet |
| Çift yönlü | Hayır | Evet (bi-directional) |
| Kullanım | HA, failover | Seçici sync, upgrade, cross-cluster |

---

## PostGIS ve Replikasyon

**Streaming Replication** PostGIS ile sorunsuz çalışır; fiziksel kopyalama yapar.

**Logical Replication** dikkat gerektiren durumlar:
- `geometry` tipi binary format — `pgoutput` plugin ile PG 15+ sürümünde desteklenir
- Yayınlamada `wal_level = logical` gereklidir
- Subscriber tarafında extension kurulu olmalıdır

## Replikasyon Mimarisi

```
Üretim (Primary):
  PostgreSQL + PostGIS
  wal_level = replica (streaming) veya logical
       |
       | WAL stream
       ↓
Yedek (Standby):
  PostgreSQL + PostGIS (aynı versiyon)
  hot_standby = on → Okuma sorguları kabul edilir

Read Replica kullanım:
  Uygulama → Primary (yazma)
  Raporlama → Standby (okuma)
```

## Yaygın Hatalar

1. **Standby read-only:** Streaming standby yazma kabul etmez. Bağlantı pool'u primary (yazma) ve standby (okuma) olarak ayrılmalıdır.
2. **PostGIS extension standby'da:** Standby fiziksel kopyadır; extension otomatik mevcuttur. Logical replication'da subscriber tarafında manuel kurulum gerekir.
3. **Replikasyon lag izleme:** `pg_stat_replication.replay_lag` kolonunun düzenli izlenmesi gerekir. Uzun lag failover süresini artırır.
