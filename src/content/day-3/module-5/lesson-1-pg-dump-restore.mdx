## Öğrenim Hedefleri

- `pg_dump` ve `pg_restore` ile yedekleme/geri yükleme stratejisini açıklamak
- PostGIS için `--no-acl --no-owner` seçeneklerini uygulamak
- Sadece geometri tablolarını yedekleyen seçici dump komutunu yazmak

---

## Yedekleme Stratejileri

| Yöntem | Komut | Boyut | Hız | Kullanım |
|---|---|---|---|---|
| SQL dump | `pg_dump -Fp` | Büyük | Yavaş | Taşıma, arşiv |
| Custom format | `pg_dump -Fc` | Küçük (sıkıştırma) | Orta | Üretim yedek |
| Directory format | `pg_dump -Fd` | Orta | Paralel | Büyük DB |
| Base backup | `pg_basebackup` | Tam disk | Hızlı | PITR, replica |

**PostGIS ek dikkat:** `geometry` tipi extension ile gelir. Geri yükleme öncesinde extension kurulu olmalıdır.

---

## Temel Yedekleme Komutları

```bash
# Tek tablo dump (sadece mekansal tablolar)
pg_dump -Fc \
  -t noktalar_perf \
  -t kaza_noktalari \
  --no-acl --no-owner \
  cbs_db > mekansal_yedek.dump

# Tüm veritabanı
pg_dump -Fc cbs_db > tam_yedek.dump

# Geri yükleme
pg_restore -d cbs_db_yeni \
  --no-acl --no-owner \
  mekansal_yedek.dump
```

**`--no-acl --no-owner`:** Yetki ve sahiplik bilgilerini dışarıda bırakır; farklı sunucuya taşımada zorunludur.

### Yedek öncesi tablo durumu

<RunnableBlock
  sql={`-- Yedek almadan önce tablo durumunu kaydet
SELECT
  relname AS tablo,
  n_live_tup AS satir_sayisi,
  pg_size_pretty(pg_total_relation_size(oid)) AS boyut
FROM pg_stat_user_tables
JOIN pg_class ON pg_class.relname = pg_stat_user_tables.relname
ORDER BY pg_total_relation_size(oid) DESC
LIMIT 10;`}
  label="Yedek öncesi tablo durumu"
  description="Yedek öncesi tablo büyüklüklerinin kaydedilmesi geri yükleme sonrası doğrulama için referans oluşturur. pg_total_relation_size tablo + indeks + toast toplam boyutunu verir. pg_size_pretty okunabilir format sağlar. n_live_tup satır sayısını gösterir; bu değer geri yükleme sonrası karşılaştırılmalıdır."

### Extension versiyonları

<RunnableBlock
  sql={`-- PostGIS extension kontrolü (geri yüklemede önce kurulmalı)
SELECT
  name,
  installed_version,
  default_version
FROM pg_available_extensions
WHERE name IN ('postgis', 'postgis_topology')
ORDER BY name;`}
  label="Extension versiyonları"
  description="pg_available_extensions mevcut ve kurulu extension versiyonlarını listeler. installed_version NULL ise extension kurulu değildir. ## Yaygın Hatalar

1. **Extension dump'a dahil edilmez:** `pg_dump` `CREATE EXTENSION postgis` komutu üretmez. Geri yükleme öncesinde `CREATE EXTENSION` manuel çalıştırılmalıdır.
2. **Büyük geometri tabloları:** `geometry` sütunu büyük veri taşır; `pg_dump` sıkıştırma (Fc format) kritik önem taşır. 1 GB tablo yaklaşık 200 MB dump'a sıkıştırılır.
3. **PITR (Point-in-Time Recovery):** SQL dump anlık snapshot'tır. Süregelen işlemler için WAL arşivleme + pg_basebackup gerekir.
