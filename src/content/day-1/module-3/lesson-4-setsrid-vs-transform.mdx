## Öğrenim Hedefleri

`ST_SetSRID` ve `ST_Transform` PostGIS'te en sık karıştırılan fonksiyonlar arasındadır. İkisi de SRID içeren bir geometri döndürür; ancak yaptıkları şey temelden farklıdır. Bu farkı kavramak, sessiz koordinat hatalarını önlemenin temelidir.

- `ST_SetSRID` ile `ST_Transform` arasındaki kesin farkı açıklayabilmek
- Yanlış kullanımın nasıl tespit edilemez hatalara yol açtığını gözlemlemek
- Doğru dönüşüm zincirini yazmak

---

## İki Farklı İşlem

### ST_SetSRID — Etiket Yapıştır

Koordinatları **değiştirmez**. Geometriye yalnızca "bu koordinatlar X sistemindedir" etiketini yapıştırır. Koordinatlar zaten doğru sistemde girilmişse kullanılır.

```sql
-- Koordinatlar aynı kalır, yalnızca SRID etiketi değişir
ST_SetSRID(geom, 4326)
```

### ST_Transform — Gerçek Koordinat Dönüşümü

Koordinatları matematiksel olarak dönüştürür. EPSG:4326'dan UTM'e geçildiğinde lon/lat değerleri metre cinsinden X/Y koordinatlarına çevrilir.

```sql
-- Koordinatlar değişir: 28.979 derece → ~500.000 metre
ST_Transform(ST_SetSRID(geom, 4326), 32636)
```

### Kullanım Kuralı

Transform çalışabilmesi için geometrinin kaynak SRID'ini bilmesi gerekir. Bu nedenle standart zincir şu sırayı izler:

```
Ham koordinatlar → ST_SetSRID(geom, kaynak_srid) → ST_Transform(..., hedef_srid)
```

### SetSRID vs Transform farkı

  SELECT ST_MakePoint(28.979, 41.015) AS raw_geom
)

  ST_AsText(ST_Transform(ST_SetSRID(raw_geom, 4326), 32636))   AS transform_utm36n,
  ST_AsText(ST_Transform(ST_SetSRID(raw_geom, 4326), 3857))    AS transform_webmercator

Bu tek sorgu iki fonksiyonun davranış farkını doğrudan gösterir."

<RunnableBlock

### 4326 → UTM → 4326 gidiş-dönüş

<RunnableBlock
  sql={`WITH istanbul_4326 AS (
  SELECT ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
),
istanbul_utm AS (
  SELECT ST_Transform(geom, 32636) AS geom FROM istanbul_4326
)
SELECT
  ST_AsText(istanbul_4326.geom)              AS baslangic_4326,
  ST_AsText(istanbul_utm.geom)               AS utm_36n,
  ST_AsText(ST_Transform(istanbul_utm.geom, 4326)) AS geri_4326
FROM istanbul_4326, istanbul_utm;`}
  label="4326 → UTM → 4326 gidiş-dönüş"
  description="İki CTE ile önce 4326→32636 dönüşümü yapılır, ardından geri dönüşüm uygulanır. Koordinatlar orijinal değerlere kayan nokta hassasiyeti sınırları içinde döner (örn. 28.9789999... yerine 28.979). Bu gidiş-dönüş tutarlılığı, dönüşüm zincirinin doğruluğunu doğrular."
/>

### UTM X/Y koordinatları

<RunnableBlock
  sql={`WITH utm AS (
  SELECT ST_Transform(
    ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326),
    32635

SELECT
  ROUND(ST_X(geom)::numeric, 0) AS utm_x_metre,

## Yaygın Hatalar

1. **ST_Transform SRID bilinmiyorsa hata verir.** Geometrinin SRID değeri -1 ise `ERROR: invalid srid` hatası alınır. `ST_SetSRID` ile önce etiketlenmesi gerekir.
2. **SetSRID dönüşüm yapmaz.** `ST_SetSRID(geom, 32636)` geometriyi UTM'e dönüştürmez; yalnızca SRID etiketini değiştirir. Bu en yaygın hatadır.
3. **Zincirleme dönüşümler geçerlidir.** `ST_Transform(ST_Transform(geom, 32636), 4326)` çalışır; her adımda SRID güncellenir.
