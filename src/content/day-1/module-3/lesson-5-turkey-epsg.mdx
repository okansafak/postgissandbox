## Öğrenim Hedefleri

Türkiye'nin coğrafi konumu birden fazla UTM dilimini kapsar; bu durum proje kapsamına göre koordinat sistemi seçimini kaçınılmaz kılar. Bu ders Türkiye'de kullanılan başlıca koordinat sistemlerini ve bölgesel dilim seçim kriterlerini ele almaktadır.

- Türkiye'de kullanılan başlıca EPSG kodlarını sıralayabilmek
- Hangi bölge için hangi UTM diliminin tercih edileceğini belirleyebilmek
- ITRF96 / TUREF ulusal datumunu tanımak

---

## Türkiye'nin Koordinat Sistemi Ortamı

Türkiye, doğu-batı doğrultusunda yaklaşık 18 derecelik açısal genişliğiyle **üç farklı UTM dilimini** kapsar. Tek bir proje tüm ülkeyi kapsıyorsa `geography` (EPSG:4326) kullanmak daha pratiktir; bölgesel hassasiyet gerektiren projelerde ise doğru UTM dilimi seçilmelidir.

### UTM Dilimleri

UTM sistemi dünyayı 6°'lik meridyen dilimlerine böler; her dilim kendi merkez meridyenine göre metre cinsinden koordinat verir.

| EPSG | Ad | Merkez Meridyen | Kapsadığı Bölge |
|---|---|---|---|
| **32635** | WGS84 / UTM Zone 35N | 27°E | Batı: İstanbul, İzmir, Bursa |
| **32636** | WGS84 / UTM Zone 36N | 33°E | Orta: Ankara, Konya, Samsun |
| **32637** | WGS84 / UTM Zone 37N | 39°E | Doğu: Erzurum, Van, Diyarbakır |

### Genel Amaçlı Sistemler

| EPSG | Ad | Kullanım |
|---|---|---|
| **4326** | WGS84 (coğrafi) | GPS, GeoJSON, API depolama |
| **3857** | Web Mercator | Google Maps, OSM tile haritaları |

### ITRF96 / TUREF — Türkiye Ulusal Datumu

Yüksek hassasiyetli kadastro ve mühendislik projelerinde resmi olarak kullanılan Türkiye datumudur.

| EPSG | Dilim |
|---|---|
| 5254 | ITRF96 / TM27 |
| 5255 | ITRF96 / TM30 |
| 5256 | ITRF96 / TM33 |
| 5257 | ITRF96 / TM36 |
| 5258 | ITRF96 / TM39 |
| 5259 | ITRF96 / TM42 |

## Uygulama Örneği

### Şehrin UTM dilimini hesapla

<RunnableBlock
  sql={`WITH sehirler AS (
  SELECT 'İstanbul' AS sehir, 28.979 AS lon, 41.015 AS lat
  UNION ALL SELECT 'Ankara',  32.854, 39.930
  UNION ALL SELECT 'İzmir',   27.143, 38.423
  UNION ALL SELECT 'Erzurum', 41.269, 39.904
  UNION ALL SELECT 'Van',     43.380, 38.494
)
SELECT
  sehir,
  lon,
  FLOOR((lon + 180) / 6) + 1                       AS utm_dilim,
  32600 + (FLOOR((lon + 180) / 6) + 1)::int        AS epsg_kodu
FROM sehirler
ORDER BY lon;`}
  label="Şehir → UTM dilim hesabı"
  description="FLOOR((lon + 180) / 6) + 1 formülü herhangi bir boylam koordinatından UTM dilim numarasını hesaplar. 32600'e ekleyerek Kuzey yarımküre için EPSG kodunu üretir. Bu hesap herhangi bir noktanın doğal UTM dilimini programatik olarak belirlemenin standart yoludur."
/>

### Aynı nokta, üç dilimde

<RunnableBlock
  sql={`WITH ankara AS (
  SELECT ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326) AS geom
)
SELECT
  ROUND(ST_X(ST_Transform(geom, 32635))::numeric) AS utm35_x,
  ROUND(ST_Y(ST_Transform(geom, 32635))::numeric) AS utm35_y,
  ROUND(ST_X(ST_Transform(geom, 32636))::numeric) AS utm36_x,
  ROUND(ST_Y(ST_Transform(geom, 32636))::numeric) AS utm36_y,
  ROUND(ST_X(ST_Transform(geom, 32637))::numeric) AS utm37_x,
  ROUND(ST_Y(ST_Transform(geom, 32637))::numeric) AS utm37_y
FROM ankara;`}
  label="Ankara — 3 UTM diliminde koordinatlar"
  description="Ankara'nın koordinatları aynı anda üç farklı UTM dilimine dönüştürülür. Doğal dilimi olan UTM 36N'de X değeri yaklaşık 500.000 metreye yakın olmalıdır — bu UTM'nin false easting (merkez meridyen ofset) özelliğidir. Diğer dilimlerde X değerleri bu referanstan uzaklaşır."
/>

### İstanbul: UTM tampon

<RunnableBlock
  sql={`WITH istanbul_utm AS (
  SELECT ST_Transform(
    ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326),
    32635
  ) AS geom
)
SELECT
  ROUND(ST_X(geom)::numeric) AS x_m,
  ROUND(ST_Y(geom)::numeric) AS y_m,
  ST_Transform(ST_Buffer(geom, 5000), 4326) AS tampon_geom
FROM istanbul_utm;`}
  label="İstanbul 5 km tampon (UTM)"
  autoFit
  description="UTM sisteminde ST_Buffer metre cinsinden çalışır: 5000 parametre 5 km tampon üretir. Tampon UTM koordinatlarında hesaplandıktan sonra ST_Transform ile 4326'ya geri çevrilerek haritada görselleştirilebilir hale getirilir. WGS84 üzerinde doğrudan ST_Buffer uygulamak derece cinsinden çalışır ve yuvarlak değil eliptik bir tampon üretir."
/>

## Yaygın Hatalar

1. **Dilim sınırında kalan noktalar:** Şehir iki dilim arasına düşüyorsa analizin ağırlıklı coğrafi bölgesine uygun dilim seçilmelidir. Türkiye geneli analizlerde `geography` (4326) tercih edilir.
2. **EPSG:3857 hassas ölçüm için uygun değildir.** Web Mercator kutuplara yaklaştıkça yaklaşık %0.3 oranında alan hatası biriktirir; yüzdesel analiz için UTM kullanılmalıdır.
3. **ITRF96 ile WGS84 farkı.** Pratik kullanımda iki datum arasındaki fark yaklaşık 1 cm'dir; çoğu uygulamada ihmal edilebilir. Kadastro hassasiyeti gerektiren projelerde EPSG:5254–5259 kullanılmalıdır.
