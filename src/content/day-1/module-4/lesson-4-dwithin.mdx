## Öğrenim Hedefleri

Yakınlık sorguları — "Bu noktaya N metre içindeki tüm nesneleri bul" — CBS uygulamalarının en yaygın analiz kalıplarından biridir. `ST_DWithin`, bu tür sorguları GiST indeks desteğiyle etkin biçimde gerçekleştiren özel bir fonksiyondur.

- `ST_DWithin` ile "N metre içindeki" geometrileri filtreleyebilmek
- `ST_DWithin` ile `ST_Distance < N` arasındaki performans farkını açıklayabilmek
- Yakınlık sorgularını doğru birimde yazmak

---

## ST_DWithin: Mesafeye Dayalı Filtre

`ST_DWithin(A, B, distance)` — A ve B arasındaki mesafe `distance` değerinden küçük veya eşit mi?

```sql
-- geography ile: distance = metre
ST_DWithin(geom_a::geography, geom_b::geography, 1000)   -- 1 km içinde

-- geometry (4326) ile: distance = derece — kaçınılmalı
ST_DWithin(geom_a, geom_b, 0.009)   -- ~1 km ama derece cinsinden
```

### ST_DWithin neden ST_Distance < N'den hızlı?

```sql
-- YAVAŞ: Her satır için tam mesafe hesaplar; indeksten yararlanamaz
WHERE ST_Distance(a.geom::geography, b.geom::geography) < 1000

-- HIZLI: GiST indeksiyle önce bounding box eleme, sonra kesin test
WHERE ST_DWithin(a.geom::geography, b.geom::geography, 1000)
```

`ST_DWithin` iki aşamalı çalışır: önce mesafe değerine göre genişletilmiş bir sınırlayıcı kutu ile aday satırları hızla elemek için GiST indeksini kullanır, ardından yalnızca adaylar üzerinde kesin geometrik test uygular.

## Uygulama Örneği

### İstanbul merkezine 100 km içindeki şehirler

<RunnableBlock
  sql={`WITH merkez AS (
  SELECT ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
),
sehirler AS (
  SELECT 'Bursa'      AS ad, ST_SetSRID(ST_MakePoint(29.061, 40.182), 4326) AS geom
  UNION ALL SELECT 'İzmit',    ST_SetSRID(ST_MakePoint(29.918, 40.766), 4326)
  UNION ALL SELECT 'Tekirdağ', ST_SetSRID(ST_MakePoint(27.508, 40.978), 4326)
  UNION ALL SELECT 'Adapazarı',ST_SetSRID(ST_MakePoint(30.399, 40.769), 4326)
  UNION ALL SELECT 'Edirne',   ST_SetSRID(ST_MakePoint(26.555, 41.677), 4326)
)
SELECT
  s.ad,
  ROUND(ST_Distance(m.geom::geography, s.geom::geography)::numeric / 1000, 0) AS km,
  ST_DWithin(m.geom::geography, s.geom::geography, 100000) AS yuz_km_icinde
FROM merkez m, sehirler s
ORDER BY km;`}
  label="İstanbul'a 100 km içindeki şehirler"
  autoFit
  description="Her şehir için iki bilgi yan yana hesaplanır: ST_Distance ile gerçek km mesafesi ve ST_DWithin ile 100 km eşiği (100.000 metre) testi. ST_DWithin boolean döndürür; WHERE koşuluna alınabilir. Sonuçlar km'ye göre sıralanarak hangi şehirlerin eşik içinde kaldığı doğrulanır."
/>

### 500 km içindeki şehir çiftleri

<RunnableBlock
  sql={`WITH sehirler AS (
  SELECT 'İstanbul'  AS ad, ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
  UNION ALL SELECT 'Ankara',   ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326)
  UNION ALL SELECT 'İzmir',    ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  UNION ALL SELECT 'Antalya',  ST_SetSRID(ST_MakePoint(30.717, 36.896), 4326)
  UNION ALL SELECT 'Trabzon',  ST_SetSRID(ST_MakePoint(39.726, 41.002), 4326)
)
SELECT
  a.ad AS sehir_1, b.ad AS sehir_2,
  ROUND(ST_Distance(a.geom::geography, b.geom::geography)::numeric / 1000, 0) AS km
FROM sehirler a, sehirler b
WHERE a.ad < b.ad
  AND ST_DWithin(a.geom::geography, b.geom::geography, 500000)
ORDER BY km;`}
  label="500 km içindeki şehir çiftleri"
  description="CROSS JOIN (FROM a, b WHERE ...) beş şehrin tüm çiftlerini oluşturur. a.ad < b.ad her çifti yalnızca bir kez listeler. ST_DWithin WHERE koşulunda 500.000 metre eşiğiyle filtreleme yapar; bu koşul büyük tablolarda GiST indeksini devreye sokar."
/>

## Alıştırma

<RunnableBlock
  sql={`WITH ankara AS (
  SELECT ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326) AS geom
),
sehirler AS (
  SELECT 'İstanbul'  AS ad, ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
  UNION ALL SELECT 'İzmir',     ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  UNION ALL SELECT 'Konya',     ST_SetSRID(ST_MakePoint(32.483, 37.872), 4326)
  UNION ALL SELECT 'Kayseri',   ST_SetSRID(ST_MakePoint(35.485, 38.720), 4326)
  UNION ALL SELECT 'Samsun',    ST_SetSRID(ST_MakePoint(36.330, 41.293), 4326)
  UNION ALL SELECT 'Eskişehir', ST_SetSRID(ST_MakePoint(30.521, 39.776), 4326)
)
SELECT s.ad,
  ROUND(ST_Distance(a.geom::geography, s.geom::geography)::numeric / 1000, 0) AS km
FROM ankara a, sehirler s
ORDER BY ST_Distance(a.geom::geography, s.geom::geography)
LIMIT 3;`}
  label="Ankara'ya en yakın 3 şehir"
  description="ORDER BY ST_Distance(...) tüm şehirleri mesafeye göre sıralar; LIMIT 3 yalnızca en yakın üçünü döndürür. Bu KNN (K-Nearest Neighbor) sorgusunun basit formudur. Büyük veri setlerinde GiST indeksli ORDER BY kNN operatörü (< - >) daha verimlidir."
/>

## Yaygın Hatalar

1. **`ST_DWithin(geometry, geometry, N)` derece kullanır.** EPSG:4326 geometrisiyle `ST_DWithin(geom, geom, 1000)` → 1000 derece — anlamsız. `::geography` cast veya UTM `ST_Transform` zorunludur.
2. **`ST_DWithin` eşit mesafeyi dahil eder.** `< distance` değil `<= distance` karşılaştırması yapar. Tam eşik değerindeki noktalar sonuca dahil olur.
3. **GiST indeksi yoksa avantaj azalır.** `CREATE INDEX USING gist(geom)` olmayan tablolarda `ST_DWithin` yine de sequential scan yapar. İndeks oluşturulduğunda kazanım dramatik olabilir.
