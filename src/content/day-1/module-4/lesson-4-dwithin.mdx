## 🎯 Ne öğreneceksin?

- `ST_DWithin` ile "N metre içindeki" geometrileri filtreleyebileceksin
- `ST_DWithin` ile `ST_Distance` arasındaki performans farkını açıklayabileceksin
- Yakınlık sorgularını GIST indeksiyle hızlandırabileceksin

---

## 📖 Kavram: Mesafeye Dayalı Filtre

`ST_DWithin(A, B, distance)` — A ve B arasındaki mesafe `distance`'tan küçük veya eşit mi?

```sql
-- geography ile: distance = metre
ST_DWithin(geom_a::geography, geom_b::geography, 1000)  -- 1 km içinde

-- geometry ile: distance = SRID birimi (4326 → derece, UTM → metre)
ST_DWithin(geom_a, geom_b, 0.009)  -- ~1 km ama derece cinsinden (kaçın!)
```

### Neden ST_DWithin > ST_Distance < ?

```sql
-- ⚠ YAVAŞ: Her satır için mesafe hesaplar, indeksten yararlanamaz
WHERE ST_Distance(a.geom::geography, b.geom::geography) < 1000

-- ✅ HIZLI: GIST indeksiyle bounding box eleme yapar, sonra kesin test
WHERE ST_DWithin(a.geom::geography, b.geom::geography, 1000)
```

## ▶ Pratik: Yakın Nokta Bulma

<RunnableBlock
  sql={`-- Birbirine 500 km'den yakın şehir çiftleri
WITH sehirler AS (
  SELECT 'İstanbul'  AS ad, ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
  UNION ALL SELECT 'Ankara',   ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326)
  UNION ALL SELECT 'İzmir',    ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  UNION ALL SELECT 'Antalya',  ST_SetSRID(ST_MakePoint(30.717, 36.896), 4326)
  UNION ALL SELECT 'Trabzon',  ST_SetSRID(ST_MakePoint(39.726, 41.002), 4326)
)
SELECT
  a.ad AS sehir_1, b.ad AS sehir_2,
  ROUND(ST_Distance(a.geom::geography, b.geom::geography)::numeric / 1000, 0) AS km
FROM sehirler a, sehirler b
WHERE a.ad < b.ad
  AND ST_DWithin(a.geom::geography, b.geom::geography, 500000)
ORDER BY km;`}
  label="500 km içindeki şehir çiftleri"

## ⚠ Tuzaklar

1. **ST_DWithin geometry'de derece kullanır:** `ST_DWithin(geom4326, geom4326, 1000)` → 1000 **derece** — anlamsız. `::geography` cast veya UTM ile kullan.
2. **ST_DWithin eşit mesafeyi dahil eder:** `< distance` değil `<= distance`. Tam sınırda olan nokta dahil.
3. **İndeks yoksa yavaş:** `CREATE INDEX USING gist(geom)` olmadan büyük tablolarda `ST_DWithin` yine de sequential scan yapar.
