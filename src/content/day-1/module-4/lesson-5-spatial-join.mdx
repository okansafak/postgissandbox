## Öğrenim Hedefleri

Mekansal JOIN, konvansiyel veritabanı JOIN mantığını geometrik ilişkilerle genişletir; "hangi şube hangi ilçede?" gibi soruları iki tablonun ortak bir ID'si olmaksızın yanıtlamayı mümkün kılar. Bu ders, mekansal JOIN'in nasıl yazıldığını ve üretim ortamları için hangi performans önlemlerinin alınması gerektiğini ele almaktadır.

- `ST_Intersects` / `ST_Contains` ile mekansal JOIN yazmak
- Nokta-poligon atamasını SQL ile gerçekleştirmek
- LEFT JOIN ile kapsaması boş poligonları raporlamak

---

## Mekansal JOIN Kavramı

Klasik JOIN iki tabloyu bir kimlik sütunu üzerinden birleştirir. Mekansal JOIN ise bu eşleştirmeyi geometrik ilişkiyle sağlar:

```sql
-- Klasik JOIN: ID üzerinden
SELECT * FROM noktalar n JOIN bolgeler b ON n.bolge_id = b.id

-- Mekansal JOIN: geometrik ilişki üzerinden
SELECT * FROM noktalar n JOIN bolgeler b ON ST_Contains(b.geom, n.geom)
```

Gerçek dünya örnekleri:
- "Hangi restoran hangi ilçede?" → `ST_Contains(ilce.geom, restoran.geom)`
- "Hangi yol hangi şehirden geçiyor?" → `ST_Intersects(sehir.geom, yol.geom)`

## Pratik Uygulama

<RunnableBlock
  sql={`WITH bolgeler AS (
  SELECT 1 AS id, 'Kuzey' AS ad, ST_SetSRID(ST_GeomFromText('POLYGON((25 41,45 41,45 45,25 45,25 41))'), 4326) AS geom
  UNION ALL SELECT 2, 'Güney', ST_SetSRID(ST_GeomFromText('POLYGON((25 36,45 36,45 38,25 38,25 36))'), 4326)
  UNION ALL SELECT 3, 'Orta',  ST_SetSRID(ST_GeomFromText('POLYGON((25 38,45 38,45 41,25 41,25 38))'), 4326)
),
sehirler AS (
  SELECT 'Ankara' AS s, ST_SetSRID(ST_MakePoint(32.854,39.930),4326) AS geom
  UNION ALL SELECT 'Konya', ST_SetSRID(ST_MakePoint(32.483,37.872),4326)
)
SELECT b.ad, COUNT(c.s) AS sehir_sayisi
FROM bolgeler b
LEFT JOIN sehirler c ON ST_Contains(b.geom, c.geom)
GROUP BY b.ad ORDER BY b.ad;`}

## Yaygın Hatalar

1. **Implicit CROSS JOIN:** `FROM a, b WHERE ST_Contains(...)` sözdizimi N×M satır üretir ve sonra filtreler. Büyük tablolarda `JOIN ... ON ST_Contains(...)` sözdizimi tercih edilmelidir.
2. **Sınır noktaları JOIN'e katılmaz.** `ST_Contains` poligon sınırı üzerindeki noktaları hariç tutar. Sınır noktalarını dahil etmek için `ST_Covers` kullanılmalıdır.
3. **GiST indeksi olmadan yavaş.** `CREATE INDEX ON tablo USING gist(geom)` yoksa büyük tablolarda mekansal JOIN sequential scan yapar ve kabul edilemez sürelere ulaşabilir.
