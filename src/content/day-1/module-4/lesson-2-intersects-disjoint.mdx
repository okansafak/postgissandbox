## 🎯 Ne öğreneceksin?

- `ST_Intersects` ile iki geometrinin kesişip kesişmediğini test edebileceksin
- `ST_Disjoint` ile tamamen ayrı geometrileri bulabileceksin
- Mekansal WHERE koşulu olarak bu fonksiyonları kullanabileceksin

---

## 📖 Kavram

### ST_Intersects(A, B)
"A ve B **ortak bir nokta paylaşıyor** mu?" — en sık kullanılan mekansal filtre.

Sınır dahil: iki poligon sadece kenarlarından dokunuyor olsa bile `true` döner.

### ST_Disjoint(A, B)
`ST_Intersects`'in tersi: "A ve B **hiç ortak nokta** paylaşmıyor mu?"

```
ST_Disjoint(A, B) = NOT ST_Intersects(A, B)
```

> **Performans:** `ST_Intersects` **GIST indeksi kullanır** — büyük tablolarda hızlı. `ST_Contains` ve diğerleri de benzer. `ST_Disjoint` indeksten yararlanamaz — büyük tablolarda dikkatli kullan.

## ▶ Canlı Örnek

<RunnableBlock
  sql={`WITH a AS (SELECT ST_SetSRID(ST_GeomFromText('POLYGON((0 0,4 0,4 4,0 4,0 0))'), 4326) AS geom),
     b AS (SELECT ST_SetSRID(ST_GeomFromText('POLYGON((2 2,6 2,6 6,2 6,2 2))'), 4326) AS geom),
     c AS (SELECT ST_SetSRID(ST_GeomFromText('POLYGON((5 5,8 5,8 8,5 8,5 5))'), 4326) AS geom),
     d AS (SELECT ST_SetSRID(ST_GeomFromText('POLYGON((4 0,6 0,6 2,4 2,4 0))'), 4326) AS geom)
SELECT
  'A-B (örtüşen)'    AS cift, ST_Intersects(a.geom,b.geom) AS intersects, ST_Disjoint(a.geom,b.geom) AS disjoint FROM a,b
UNION ALL SELECT 'A-C (ayrı)',  ST_Intersects(a.geom,c.geom), ST_Disjoint(a.geom,c.geom) FROM a,c
UNION ALL SELECT 'A-D (sınır)', ST_Intersects(a.geom,d.geom), ST_Disjoint(a.geom,d.geom) FROM a,d;`}
  label="Intersects ve Disjoint karşılaştırması"
/>

## ▶ Gerçek Senaryo — Bölgeyle Kesişen Rotalar

<RunnableBlock
  sql={`WITH istanbul_bolge AS (
  SELECT ST_SetSRID(
    ST_GeomFromText('POLYGON((28.5 40.8, 29.5 40.8, 29.5 41.3, 28.5 41.3, 28.5 40.8))'),
    4326) AS geom
),
rotalar AS (
  SELECT 'İstanbul-Ankara'  AS ad, ST_SetSRID(ST_GeomFromText('LINESTRING(28.979 41.015, 32.854 39.930)'), 4326) AS geom
  UNION ALL
  SELECT 'İzmir-Ankara',         ST_SetSRID(ST_GeomFromText('LINESTRING(27.143 38.423, 32.854 39.930)'), 4326)

  SELECT 'Edirne-Tekirdağ',      ST_SetSRID(ST_GeomFromText('LINESTRING(26.555 41.678, 27.508 40.977)'), 4326)
)
SELECT
  r.ad,
  ST_Intersects(b.geom, r.geom) AS istanbul_bolgesinden_geciyor
FROM istanbul_bolge b, rotalar r;`}

## 🧪 Dene

Tamamen ayrı (disjoint) geometrileri filtrele:

<RunnableBlock
  sql={`WITH referans AS (
  SELECT ST_SetSRID(ST_GeomFromText('POLYGON((0 0,5 0,5 5,0 5,0 0))'), 4326) AS geom
),
test AS (
  SELECT 1 AS id, ST_SetSRID(ST_GeomFromText('POLYGON((3 3,7 3,7 7,3 7,3 3))'), 4326) AS geom
  UNION ALL SELECT 2, ST_SetSRID(ST_GeomFromText('POLYGON((6 6,9 6,9 9,6 9,6 6))'), 4326)
  UNION ALL SELECT 3, ST_SetSRID(ST_GeomFromText('POINT(2 2)'), 4326)
  UNION ALL SELECT 4, ST_SetSRID(ST_GeomFromText('POINT(8 8)'), 4326)
)
SELECT t.id, ST_GeometryType(t.geom) AS tip,
  ST_Intersects(r.geom, t.geom) AS kesiyor
FROM referans r, test t
ORDER BY t.id;`}
  label="Kesişim filtresi"
/>

## ⚠ Tuzaklar

1. **ST_Disjoint büyük tablolarda yavaş:** `WHERE ST_Disjoint(a, b)` indeks kullanamaz. Bunun yerine `WHERE NOT ST_Intersects(a, b)` yaz — aynı sonuç, indeksli.
2. **Sadece sınır teması:** İki poligon sadece bir kenar noktasıyla temas ediyorsa `ST_Intersects` → `true`. Yüzey örtüşmesi için `ST_Overlaps` kullan.
3. **Geometri tipini karıştırma:** `ST_Intersects(point, point)` yalnızca aynı koordinatta `true` döner.
