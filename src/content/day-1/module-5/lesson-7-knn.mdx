## Öğrenim Hedefleri

"En yakın N nokta" sorgusu — sağlık hizmetlerine erişimden acil servis yönlendirmesine kadar geniş bir uygulama yelpazesinde ortaya çıkar. Bu ders, GiST indeksini ORDER BY'da kullanan `<->` operatörünün nasıl çalıştığını ve LATERAL JOIN ile her satır için en yakın komşunun nasıl bulunduğunu ele almaktadır.

- `<->` KNN operatörü ile en yakın N komşuyu verimli bulmak
- `ORDER BY ... <-> ... LIMIT N` kalıbını uygulamak
- LATERAL JOIN ile her satır için en yakın komşuyu hesaplamak

---

## KNN: K-Nearest Neighbor

`<->` operatörü iki geometrinin **bbox merkezlerine göre yaklaşık mesafeyi** hesaplar. Asıl önemi, büyük tablolarda sıralı arama için GiST indeksini devreye sokabilmesidir.

```sql
-- En yakın 5 nokta — GiST indeks ile hızlı
SELECT id, ST_Distance(a.geom::geography, ref.geom::geography)
FROM noktalar a, referans ref
ORDER BY a.geom <-> ref.geom
LIMIT 5;
```

### `ST_DWithin` ile birleştirme

Çok büyük tablolarda önce `ST_DWithin` ile coğrafi bir pencere açılır, ardından KNN sıralaması yapılır:

```sql
WHERE ST_DWithin(a.geom::geography, ref.geom::geography, 100000)  -- 100 km içinde
ORDER BY a.geom <-> ref.geom
LIMIT 5
```

### Ankara'ya en yakın 3 şehir (KNN)

<RunnableBlock
  sql={`WITH referans AS (
  SELECT ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326) AS geom
),
sehirler AS (
  SELECT 'İstanbul'  AS ad, ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
  UNION ALL SELECT 'İzmir',    ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  UNION ALL SELECT 'Konya',    ST_SetSRID(ST_MakePoint(32.483, 37.872), 4326)
  UNION ALL SELECT 'Kayseri',  ST_SetSRID(ST_MakePoint(35.485, 38.720), 4326)
  UNION ALL SELECT 'Samsun',   ST_SetSRID(ST_MakePoint(36.330, 41.293), 4326)
  UNION ALL SELECT 'Eskişehir',ST_SetSRID(ST_MakePoint(30.521, 39.776), 4326)
  UNION ALL SELECT 'Trabzon',  ST_SetSRID(ST_MakePoint(39.726, 41.002), 4326)
)
SELECT
  s.ad,
  ROUND(ST_Distance(s.geom::geography, r.geom::geography)::numeric / 1000, 0) AS km,
  s.geom
FROM sehirler s, referans r
ORDER BY s.geom <-> r.geom
LIMIT 3;`}

ST_Distance doğrulama amacıyla gerçek elipsoid mesafesini km cinsinden hesaplar. Gerçek tablolarda GiST indeksi bu sıralamayı tam tablo taraması olmadan gerçekleştirir."

### Her şehrin en yakın komşusu (LATERAL JOIN)

<RunnableBlock
  sql={`WITH sehirler AS (
  SELECT 'İstanbul'  AS ad, ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
  UNION ALL SELECT 'Ankara',   ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326)
  UNION ALL SELECT 'İzmir',    ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  UNION ALL SELECT 'Konya',    ST_SetSRID(ST_MakePoint(32.483, 37.872), 4326)
  UNION ALL SELECT 'Kayseri',  ST_SetSRID(ST_MakePoint(35.485, 38.720), 4326)
)
SELECT
  a.ad AS sehir,
  knn.ad AS en_yakin,
  ROUND(ST_Distance(a.geom::geography, knn.geom::geography)::numeric / 1000, 0) AS km
FROM sehirler a,
LATERAL (
  SELECT b.ad, b.geom
  FROM sehirler b
  WHERE b.ad <> a.ad
  ORDER BY a.geom <-> b.geom
  LIMIT 1
) knn;`}
  label="Her şehrin en yakın komşusu"
  description="LATERAL alt sorgu dış sorgunun her satırı (a) için bağımsız olarak çalışır: o satırın geometrisine en yakın diğer şehri bulur. WHERE b.ad <> a.ad şehrin kendisiyle eşleşmesini önler. Bu kalıp — dış döngü + LATERAL KNN — her nokta için en yakın komşuyu paralel biçimde hesaplamanın standart yöntemidir."
/>

## Yaygın Hatalar

1. **`<->` kesin geometri mesafesi değildir.** Bbox merkezleri arasındaki yaklaşık mesafeyi kullanır. Küçük nokta geometrilerinde fark ihmal edilebilir; büyük poligonlarda sapma olabilir. Kesin sıralama için `ST_Distance` ile doğrulama yapılmalıdır.
2. **GiST indekssiz tabloda KNN avantajı yoktur.** `<->` operatörü `CREATE INDEX USING gist(geom)` olmayan tablolarda `ST_Distance` kadar yavaş çalışır. İndeks oluşturulduğunda kazanım dramatik olabilir.
3. **N-N eşleştirme için LATERAL zorunludur.** Her satır için en yakın komşuyu bulmak istenen durumlarda `LATERAL` olmadan yazılan sorgu CROSS JOIN üretir ve doğru sonuç vermez.
