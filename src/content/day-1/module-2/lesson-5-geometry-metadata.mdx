## Öğrenim Hedefleri

Geometri ile çalışırken tipi, koordinat sistemini, boyutunu ve yapısal özelliklerini sorgulamak analitik sürecin ayrılmaz bir parçasıdır. Bu ders PostGIS'in introspection (iç yapı sorgulama) fonksiyonlarını ve sınırlayıcı kutu hesaplamalarını kapsamaktadır.

- `ST_NPoints`, `ST_NumGeometries` ile yapısal bilgi çekmek
- `ST_SRID`, `ST_CoordDim`, `ST_GeometryType` fonksiyonlarını kullanmak
- `ST_Envelope` ile sınırlayıcı dikdörtgen hesaplamak

---

## Geometri Introspection Fonksiyonları

Bir geometri nesnesinin iç yapısını sorgulamak için kullanılan temel fonksiyonlar:

| Fonksiyon | Açıklama | Örnek Çıktı |
|---|---|---|
| `ST_GeometryType(g)` | OGC tip adı | `ST_LineString` |
| `ST_SRID(g)` | Koordinat sistemi kodu | `4326` |
| `ST_CoordDim(g)` | Koordinat boyutu | `2` veya `3` |
| `ST_NPoints(g)` | Toplam nokta sayısı | `5` |
| `ST_NumGeometries(g)` | MULTI* içindeki parça sayısı | `3` |
| `ST_IsEmpty(g)` | Boş geometri mi? | `false` |
| `ST_Envelope(g)` | Sınırlayıcı dikdörtgen (POLYGON) | `POLYGON((...))` |
| `ST_Extent(g)` | Aggregate sınırlayıcı kutu | `BOX(...)` |

## Uygulama Örneği

### Tek geometrinin tam metadata özeti

<RunnableBlock
  sql={`WITH geom AS (
  SELECT ST_SetSRID(
    ST_GeomFromText('LINESTRING(28.9 41.0, 29.0 41.0, 29.2 40.8, 29.5 40.6, 30.0 40.3)'),
    4326
  ) AS g
)
SELECT
  ST_GeometryType(g)         AS tip,
  ST_SRID(g)                 AS srid,
  ST_CoordDim(g)             AS boyut,
  ST_NPoints(g)              AS nokta_sayisi,
  ST_IsEmpty(g)              AS bos_mu,
  ST_AsText(ST_Envelope(g))  AS envelope_wkt
FROM geom;`}
  label="Geometri metadata özeti"
  description="Tek bir CTE içinde oluşturulan LINESTRING üzerinde altı farklı introspection fonksiyonu uygulanır. ST_CoordDim 2D/3D ayrımını, ST_NPoints tüm segmentlerin köşe noktası sayısını, ST_Envelope ise geometriyi tamamen kapsayan minimum dikdörtgeni döndürür."
/>

### ST_Envelope: Sınırlayıcı kutu

<RunnableBlock
  sql={`WITH sehirler AS (
  SELECT ST_Collect(ARRAY[
    ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326),
    ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326),
    ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  ]) AS geom
)
SELECT
  ST_AsText(ST_Envelope(geom)) AS envelope,
  ST_AsText(geom)              AS noktalar
FROM sehirler;`}
  label="3 şehrin sınırlayıcı kutusu"
  autoFit
  description="ST_Collect önce üç noktayı MULTIPOINT'te birleştirir. ST_Envelope bu koleksiyonu kapsayan minimum eksen-hizalı dikdörtgeni (POLYGON) döndürür. Harita zoom seviyesini otomatik ayarlamak veya mekansal indeks sorgularını optimize etmek için kullanılır."
/>

## Pratik Uygulama

### Aggregate bounding box

<RunnableBlock
  sql={`WITH noktalar AS (
  SELECT ST_SetSRID(ST_MakePoint(28.979, 41.015), 4326) AS geom
  UNION ALL SELECT ST_SetSRID(ST_MakePoint(32.854, 39.930), 4326)
  UNION ALL SELECT ST_SetSRID(ST_MakePoint(27.143, 38.423), 4326)
  UNION ALL SELECT ST_SetSRID(ST_MakePoint(36.336, 37.008), 4326)
)
SELECT ST_Extent(geom)::text AS turkiye_bbox
FROM noktalar;`}
  label="Aggregate bounding box"
  description="ST_Extent bir aggregate fonksiyondur: GROUP BY gibi birden fazla satırdaki geometrilerin tamamını kapsayan en küçük dikdörtgeni tek bir BOX değeri olarak döndürür. ST_Envelope'dan farkı, ayrı satırlardaki geometriler üzerinde çalışmasıdır."
/>

### MULTIPOLYGON metadata

<RunnableBlock
  sql={`WITH mp AS (
  SELECT ST_GeomFromText('MULTIPOLYGON(
    ((0 0, 1 0, 1 1, 0 1, 0 0)),
    ((2 0, 3 0, 3 1, 2 1, 2 0)),
    ((4 0, 5 0, 5 1, 4 1, 4 0))
  )') AS geom
)
SELECT
  ST_NumGeometries(geom) AS parca_sayisi,
  ST_NPoints(geom)       AS toplam_nokta,
  ST_GeometryType(geom)  AS tip
FROM mp;`}
  label="MULTIPOLYGON metadata"
  description="Üç ayrı POLYGON içeren bu MULTIPOLYGON üzerinde ST_NumGeometries 3 döndürür. ST_NPoints tüm parçaların tüm köşe noktalarını toplar: her kare poligon 5 nokta (kapama dahil) içerdiğinden toplam 15 olmalıdır."
/>

## Yaygın Hatalar

1. **ST_NPoints MULTI tiplerinde tüm parçaları sayar.** `MULTILINESTRING` içindeki her segmentin tüm noktaları toplanır; yalnızca ilk parçanın sayısı değildir.
2. **ST_Extent ve ST_Envelope karıştırılmamalıdır.** `ST_Extent` birden fazla satırın aggregate toplamıdır; `ST_Envelope` tek bir geometri üzerinde çalışır.
3. **3D geometriler:** `ST_CoordDim` → 3 sonucu Z koordinatı bulunduğunu gösterir. Z değerine `ST_Z(point)` ile erişilir; 2D fonksiyonlar Z boyutunu yok sayar.
